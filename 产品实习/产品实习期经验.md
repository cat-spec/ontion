# 项目经验
## 项目一 IM管理系统
负责用户画像的制作
用户列表->用户标签->用户分群->用户画像
在IM（即时通讯）产品中，这四个页面不是孤立的，而是一个**从“找到人”到“看清人”，再到“把人聚拢运营”的闭环**。结合IM场景，它们的联动关系如下：
**1. 用户列表（起点与终点）**
- 展示所有IM联系人或客户会话列表（客服工作台、销售侧边栏等）。

- 在这里可以快速查看每个用户的关键标签和最近互动状态。
    
- **联动动作**：
    
    - 选中一个用户 → 进入**用户画像**，查看详情。
        
    - 勾选多个用户 → 批量打**标签**。
        
    - 列表的筛选和排序条件，可以直接调用**标签**和**分群**的结果（例如“只看标签为‘高意向’的用户”“只看‘沉默客户’分群”）。
        

**2. 用户标签（定义用户属性的积木）**

- 标签来源：手动打标、根据IM对话关键词自动打标（如客户提到“退费”自动加“售后风险”）、从行为打标（如“近30天无消息”）。
    
- **联动核心**：
    
    - 所有标签会直接映射到**用户画像**中，成为画像维度。
        
    - 标签是**分群**的原子条件——分群就是“标签 + 行为 + 属性”的组合。
        
    - 在标签管理页面，可以查看“每个标签覆盖多少用户”，点击数字直接进入筛选后的**用户列表**。
        

**3. 用户分群（动态组合与运营对象）**

- 基于标签、IM行为（如消息数、响应时长、会话发起次数）等条件，生成动态或静态群组，例如“跨境新客户且使用翻译功能多次”。
    
- **联动体现**：
    
    - 分群创建后，在**用户列表**可一键筛选出该群体，进行群发、批量操作。
        
    - 分群可跳转查看**群体画像**（聚合数据），看到这群人的共同特征、活跃时段、翻译语言偏好等。
        
    - 分群内的个体，可快速进入**单用户画像**，进行1对1跟进。
        

**4. 用户画像（全景洞察与动作入口）**

- 分为**单用户画像**和**群体画像**。
    
    - 单用户画像：侧边栏式的全景卡片（在IM对话右侧），融合标签、分群归属、历史订单、翻译记录、互动趋势等。
        
    - 群体画像：针对某个分群的统计仪表盘。
        
- **闭环操作**：
    
    - 在画像中可以直接增删**标签**，系统自动刷新该用户的分群归属。
        
    - 画像里点击某个标签，会显示“同样有此标签的用户”，进入**用户列表**。
        
    - 画像中展示该用户所处的**分群**，点击可查看该分群的整体画像。
        
    - 对群体画像分析后，可以反向拆解出新的**标签**规则或**分群**条件，回到标签或分群页面优化。
        

**IM场景特有的联动价值**：  
因为IM承载的是实时对话，所以这四个页面可以做到**即时反馈**。比如客服在和客户聊天时，从IM侧边栏看到画像缺少某个标签，立刻补打；打标后，该客户可能立刻进入“高意向客户分群”，并触发自动欢迎语或SOP，这就形成了实时运营联动。

## 项目二 scrm跨境翻译
- **项目名称**：跨境多平台SCRM翻译工作台（桌面端+商户后台）
- **我的角色**：产品实习生（负责竞品调研、AI原型设计、需求输出及部分交互逻辑）
- **解决问题**：跨境商家在WhatsApp、Messenger等多平台与买家沟通时，语言障碍导致回复慢、错译多、客户流失。
- **解决方案**：将多平台消息统一接入SCRM后台，内嵌智能翻译引擎，支持自动/手动翻译、翻译记忆库、人工校对，并关联客户画像，让语言服务成为客户运营的一环。
- **核心成果**：输出完整原型并进入开发，原型设计周期借助AI缩短约40%；分享AI工作流推动团队效率提升。
**我的设计逻辑提炼**：  
不做通用翻译工具，而是做**生长在IM对话里的翻译能力**。翻译不仅解决语言转换，还要成为客户洞察的数据源——比如通过翻译内容分析客户意图，自动打上“询价”“售后”等标签，让翻译功能成为SCRM客户分层的一部分，实现线索，底粉，客户全流程闭环。

我们不直接书写prd需求文档，直接让开发看懂需求设计与界面联动，直接将做好的前端原形进行标注处理。

#### 系统功能
1.核心翻译机制 截获webview的消息框、发送框等等dom元素，直接进行dom注入，来实现翻译功能。
2.欢迎语，自动回复，关键词回复三重转化机制。
3.客户标签，客户画像手动上报机制，后台清晰看到客户类型。
4.多账号多开机制，线索、底粉、客户 去重机制，保障线索优质无重复。
5.引流分流机制，1对1，1对多进行加粉快捷连接。
