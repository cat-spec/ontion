# IM后台风险异常统计需求文档

# 一、风险异常统计

### 1.1 文档说明

本文档用于说明 IM 后台「统计管理 / 风险异常统计」页面的产品需求，面向产品、前端、测试、运营、客服、风控等角色统一理解页面目标、展示结构、指标口径、交互规则和验收标准。

本文档基于当前已实现页面和最终页面效果编写，不包含后端接口设计、数据库设计、权限点设计和真实风控规则算法。当前页面数据用于前端展示验证，后续接入真实数据时应保持本文档定义的展示结构和统计口径。

### 1.2 模块定位

风险异常统计用于展示 IM 平台在指定周期内的异常事件规模、风险对象规模、异常类型结构和每日明细数据，帮助风控、运营和管理人员快速判断平台风险变化趋势，并定位主要异常来源。

页面核心需要回答以下问题：

| 问题 | 对应展示 |
| --- | --- |
| 当前周期异常规模有多大 | 顶部统计卡片：异常事件数 |
| 当前周期影响了多少风险对象 | 顶部统计卡片：风险用户数、风险 IP 数、风险设备数 |
| 强管控对象规模如何 | 顶部统计卡片：黑名单 IP 数 |
| 异常事件是否持续升高 | 风险异常趋势图 |
| 主要异常类型是什么 | 异常类型占比环形图 |
| 每日、每端的风险数据如何分布 | 风险异常明细表 |

### 1.3 适用角色

| 角色 | 使用目标 |
| --- | --- |
| 风控人员 | 判断异常行为规模、识别主要异常类型、定位高风险日期 |
| 运营人员 | 观察风险事件对用户运营的影响，辅助评估活动或用户增长带来的风险变化 |
| 客服人员 | 理解投诉、账号异常、设备异常等风险背景，辅助用户问题排查 |
| 管理人员 | 快速查看风险规模和趋势，判断是否需要升级处理策略 |
| 测试人员 | 根据需求口径验证页面展示、筛选、图表和表格逻辑 |

### 1.4 页面结构

页面由四个区域组成。

| 区域 | 内容 | 说明 |
| --- | --- | --- |
| 顶部筛选区 | 时间快捷筛选、自定义日期范围、查询、重置 | 控制顶部统计卡片和核心可视化区域 |
| 第一排统计卡片 | 风险用户数、黑名单 IP 数、风险 IP 数、风险设备数、异常事件数 | 展示当前周期核心汇总指标 |
| 第二排核心可视化区域 | 风险异常趋势图、异常类型占比环形图 | 展示风险趋势和异常类型结构 |
| 第三排明细区 | 风险异常明细标题、工具筛选区、明细表格 | 展示每日、每端风险异常明细 |

页面默认统计近 7 日数据。顶部筛选区不计入三排布局。

### 1.5 顶部筛选区

顶部筛选区用于控制页面核心汇总数据和核心可视化图表的统计周期。

| 筛选项 | 说明 |
| --- | --- |
| 今日 | 统计当天风险异常数据 |
| 昨日 | 统计昨日风险异常数据 |
| 近 7 日 | 默认选项，统计最近 7 日风险异常数据 |
| 近 30 天 | 统计最近 30 天风险异常数据 |
| 自定义日期范围 | 用户手动选择开始日期和结束日期 |
| 查询 | 按当前日期范围刷新统计卡片和核心图表 |
| 重置 | 恢复默认近 7 日 |

交互规则：

1. 点击快捷时间后，自动回填日期范围。
2. 手动调整日期范围后，应按用户选择的开始日期和结束日期查询。
3. 点击查询后，刷新顶部统计卡片、风险异常趋势图和异常类型占比图。
4. 点击重置后，恢复默认近 7 日。
5. 明细区拥有独立工具筛选区，默认同步顶部日期范围，但后续可单独筛选。

### 1.6 顶部统计卡片

顶部统计卡片展示当前统计周期内的核心汇总指标。

| 卡片 | 主指标 | 底部摘要 | 说明 |
| --- | --- | --- | --- |
| 风险用户数 | 当前周期风险用户数 | 人均异常次数、高风险用户占比 | 体现异常事件影响到的去重用户规模 |
| 黑名单 IP 数 | 当前周期黑名单 IP 数 | 风险 IP 中强管控占比 | 体现已进入强管控状态的 IP 规模 |
| 风险 IP 数 | 当前周期风险 IP 数 | IP 异常占全部事件比例 | 体现 IP 维度的风险规模 |
| 风险设备数 | 当前周期风险设备数 | 设备异常占全部事件比例 | 体现设备维度的风险规模 |
| 异常事件数 | 当前周期异常事件数 | 主要异常类型、影响用户数 | 体现系统捕获到的异常行为总量 |

展示规则：

1. 每张卡片展示指标标题、主数值、底部摘要和辅助图标。
2. 主数值使用千分位展示。
3. 底部摘要应优先表达业务判断，不应只重复主数值。
4. 顶部卡片用于快速判断风险规模、风险对象和主要异常来源。

### 1.7 异常与风险口径说明

风险异常统计页面中，「异常」和「风险」属于不同统计口径。

异常是事件口径，描述系统捕获到多少次可疑行为或异常命中。异常事件允许同一用户、同一 IP、同一设备在同一统计周期内多次命中，因此异常事件数可以重复累计。

风险是对象口径，描述多少用户、IP 或设备因异常行为被判定为存在风险。风险对象通常按用户、IP 或设备去重统计。

| 类型 | 口径 | 是否允许重复 | 示例 |
| --- | --- | --- | --- |
| 异常事件数 | 事件口径 | 允许重复 | 同一用户一天命中 3 次屏蔽词，计 3 次异常事件 |
| 风险用户数 | 用户对象口径 | 去重 | 同一用户一天命中多次异常，风险用户数通常计 1 人 |
| 风险 IP 数 | IP 对象口径 | 去重 | 同一 IP 多次触发异常，风险 IP 数通常计 1 个 |
| 风险设备数 | 设备对象口径 | 去重 | 同一设备多次触发异常，风险设备数通常计 1 台 |

### 1.8 指标统计口径

| 指标 | 统计口径 |
| --- | --- |
| 异常事件数 | 统计周期内所有异常类型命中次数之和，允许同一对象重复命中 |
| 风险用户数 | 统计周期内命中过异常并被归入风险池的去重用户数 |
| 高风险用户数 | 风险用户中风险等级为高的去重用户数 |
| 中风险用户数 | 风险用户中风险等级为中的去重用户数 |
| 低风险用户数 | 风险用户中风险等级为低的去重用户数 |
| 风险层级分布 | 高风险用户数、中风险用户数、低风险用户数组成的分布 |
| 风险 IP 数 | 统计周期内命中过异常规则的去重 IP 数 |
| 黑名单 IP 数 | 风险 IP 中已进入黑名单或强管控状态的 IP 数 |
| 风险设备数 | 统计周期内命中过异常规则的去重设备数 |
| 设备异常 | 设备维度异常命中次数 |
| IP 异常 | IP 维度异常命中次数 |
| 屏蔽词命中 | 消息内容命中屏蔽词或敏感词的次数 |
| 账号异常 | 登录、注册、账号状态等账号维度异常命中次数 |
| 高频加好友 | 短时间内频繁发起加好友行为的异常次数 |
| 高频群发 | 短时间内频繁群发或批量发送相似内容的异常次数 |
| 主要异常类型 | 当日或当前周期内异常事件数最高的异常类型 |

风险层级分布规则：

1. 风险层级分布按用户数统计。
2. 高风险用户数、中风险用户数、低风险用户数应合并展示为一个堆叠进度条。
3. 三类风险用户数相加应等于或接近风险用户数。
4. 鼠标悬停堆叠进度条时，应展示高中低风险用户数明细。

### 1.9 核心可视化区域

核心可视化区域位于页面第二排，包含左侧风险异常趋势图和右侧异常类型占比图。

#### 1.9.1 风险异常趋势图

风险异常趋势图用于展示统计周期内异常事件和风险对象的变化趋势。

图表类型为双柱状 + 双折线组合图。

| 指标 | 图形类型 | 说明 |
| --- | --- | --- |
| 异常事件数 | 柱状图 | 每日异常事件总数 |
| 异常用户数 | 柱状图 | 每日命中异常的风险用户数 |
| 风险 IP 数 | 折线图 | 每日风险 IP 数 |
| 风险设备数 | 折线图 | 每日风险设备数 |

展示规则：

1. 横轴按日期展示。
2. 左侧 Y 轴展示异常事件数和异常用户数。
3. 右侧 Y 轴展示风险 IP 数和风险设备数。
4. 图例展示异常事件数、异常用户数、风险 IP 数、风险设备数。
5. 图例支持点击控制系列显隐。
6. Hover 时展示当日完整数据。

卡片底部展示自动结论，示例：

```text
异常峰值出现在 2026-06-02，异常事件 426，异常用户 279
```

该结论用于快速定位统计周期内风险异常峰值日期。

#### 1.9.2 异常类型占比

异常类型占比用于展示当前统计周期内不同异常类型的结构分布。

图表类型为环形图。

异常类型包括：

| 异常类型 | 说明 |
| --- | --- |
| 设备异常 | 设备复用、设备波动、异常设备行为等 |
| IP 异常 | 高频 IP、风险 IP、异常地区 IP 等 |
| 屏蔽词命中 | 消息内容命中屏蔽词或敏感词 |
| 账号异常 | 登录、注册、账号状态等异常 |
| 高频加好友 | 短时间内频繁发起加好友 |
| 高频群发 | 短时间内批量发送或群发相似内容 |

展示规则：

1. 环形图按异常事件数计算占比。
2. 图例展示全部异常类型。
3. Hover 时展示异常类型、次数和占比。
4. 卡片底部展示当前周期占比最高的异常类型。

底部结论示例：

```text
当前周期占比最高：设备异常，共 602 次
```

### 1.10 风险异常明细

风险异常明细位于页面第三排，用于按日期和终端展示风险异常明细数据，并支持局部筛选和排序。

#### 1.10.1 区域结构

| 区域 | 内容 |
| --- | --- |
| 标题 | 风险异常明细 |
| 工具筛选区 | 日期范围、异常类型、终端类型、查询、重置 |
| 明细表格 | 每日、每端风险异常数据 |

标题下不展示摘要文案，避免与顶部统计卡片和图表结论重复。

#### 1.10.2 工具筛选区

| 筛选项 | 说明 |
| --- | --- |
| 日期范围 | 查询明细数据的统计日期范围 |
| 异常类型 | 支持设备异常、IP 异常、屏蔽词命中、账号异常、高频加好友、高频群发 |
| 终端类型 | 支持 Android、iOS |
| 查询 | 按当前工具筛选条件刷新明细表 |
| 重置 | 恢复当前顶部日期范围，并清空异常类型和终端类型 |

筛选规则：

1. 异常类型不提供“全部”选项，未选择时表示全部异常类型。
2. 终端类型不提供“全部”选项，未选择时表示全部终端。
3. 日期范围未选择时，默认按近 7 日查询。
4. 所有筛选框通过 placeholder 表达默认全部含义。
5. 工具筛选区删除风险层级和关键字筛选。

#### 1.10.3 明细表字段

| 字段 | 说明 |
| --- | --- |
| 统计日期 | 数据所属日期 |
| 异常事件数 | 当日该终端异常事件总数 |
| 风险用户数 | 当日该终端命中风险的去重用户数 |
| 风险层级分布 | 高、中、低风险用户数合并展示为堆叠进度条 |
| 黑名单 IP 数 | 当日该终端黑名单 IP 数 |
| 风险 IP 数 | 当日该终端风险 IP 数 |
| 风险设备数 | 当日该终端风险设备数 |
| 设备异常 | 当日该终端设备异常次数 |
| IP 异常 | 当日该终端 IP 异常次数 |
| 屏蔽词命中 | 当日该终端屏蔽词命中次数 |
| 账号异常 | 当日该终端账号异常次数 |
| 高频加好友 | 当日该终端高频加好友异常次数 |
| 高频群发 | 当日该终端高频群发异常次数 |
| 主要异常类型 | 当日该终端异常次数最高的异常类型 |

字段顺序要求：

1. 异常事件数应放在风险用户数前面。
2. 风险层级分布不拆成高风险用户数、中风险用户数、低风险用户数三个独立字段。
3. 不展示闭环率、已关闭事件、平均处置、待处理事件数和高风险事件数。

#### 1.10.4 风险层级分布展示规则

风险层级分布使用堆叠进度条展示。

展示内容：

```text
高风险 272 / 中风险 556 / 低风险 700
[ 红色段 ][ 橙色段 ][ 绿色段 ]
```

颜色规则：

| 风险层级 | 颜色 |
| --- | --- |
| 高风险 | 红色 |
| 中风险 | 橙色 |
| 低风险 | 绿色 |

交互规则：

1. 表格内默认展示简短数字和堆叠进度条。
2. 鼠标悬停时展示完整 tooltip。
3. tooltip 展示高风险用户数、中风险用户数、低风险用户数。
4. 当风险用户数为 0 时，进度条可置灰或不展示。

#### 1.10.5 排序规则

列表支持核心数值字段排序。

支持排序字段：

| 字段 | 排序规则 |
| --- | --- |
| 统计日期 | 按日期升序或降序 |
| 异常事件数 | 按数值升序或降序 |
| 风险用户数 | 按数值升序或降序 |
| 黑名单 IP 数 | 按数值升序或降序 |
| 风险 IP 数 | 按数值升序或降序 |
| 风险设备数 | 按数值升序或降序 |
| 各异常类型字段 | 按数值升序或降序 |

### 1.11 页面默认状态

页面默认状态如下：

| 项目 | 默认值 |
| --- | --- |
| 顶部时间范围 | 近 7 日 |
| 明细日期范围 | 默认同步顶部时间范围 |
| 明细异常类型 | 未选择，表示全部 |
| 明细终端类型 | 未选择，表示全部 |
| 明细排序 | 默认按统计日期倒序 |
| 明细分页 | 使用系统默认分页配置 |

### 1.12 数据展示优化要求

页面应满足以下产品使用要求：

1. 顶部统计卡片底部摘要应体现判断价值，不应只重复主指标。
2. 风险用户数卡片需展示人均异常次数和高风险用户占比。
3. 黑名单 IP 数卡片需展示风险 IP 中强管控占比。
4. 风险 IP 数卡片需展示 IP 异常在全部事件中的占比。
5. 风险设备数卡片需展示设备异常在全部事件中的占比。
6. 异常事件数卡片需展示主要异常类型和影响用户数。
7. 风险异常趋势图应默认展示全部四个指标，图例可用于控制显隐。
8. 异常类型占比图需展示类型结构和最高占比类型结论。
9. 明细表需要支持风控人员继续下钻分析每日风险结构。
10. 页面整体视觉风格需与用户增长统计、转化统计、活跃统计保持一致。

### 1.13 不包含范围

当前版本不包含以下内容：

1. 不包含后端接口设计。
2. 不包含真实风控规则算法。
3. 不包含风险处置闭环、待处理、已关闭、平均处置时长等处置效率指标。
4. 不包含关键字搜索。
5. 不包含风险层级筛选。
6. 不包含导出功能。
7. 不包含跳转到风险用户、风险 IP、风险设备详情页。

| 验收项 | 验收标准 |
| --- | --- |
| 页面入口 | 统计管理下存在风险异常统计页面 |
| 默认展示 | 页面默认展示近 7 日数据 |

| 环形图 | 展示 6 类异常类型占比 |
| 明细筛选 | 支持日期范围、异常类型、终端类型筛选 |
| 筛选默认态 | 异常类型和终端类型未选择时表示全部，不展示“全部”选项 |

### 1.15 后续扩展建议

后续接入真实数据后，可根据业务需要扩展以下能力：

1. 增加较上周期变化，用于判断风险是否快速上升。
2. 增加点击图表联动明细列表能力。
3. 增加风险用户、风险 IP、风险设备的详情跳转。
4. 增加导出功能，支持风控复盘。
5. 增加风险处置效率统计，但应作为独立处置分析模块，不建议混入当前风险异常统计主页面。
