# IM后台活跃统计需求文档

## 一、文档说明

本文档用于说明 IM 后台「活跃统计」页面的产品需求，统一产品、前端、后端、测试、运营对页面目标、指标口径、数据展示和验收标准的理解。

本页面归属统计管理模块，重点解决“活跃用户规模如何”“活跃是否形成真实互动”“用户池结构是否健康”“近期活跃趋势是否改善”四类问题。

本文档当前版本只定义页面展示、指标口径、交互逻辑和验收要求，不包含后端接口设计、数据库设计及活跃分层判定规则引擎实现细节。

## 二、产品目标

活跃统计页面用于衡量平台用户活跃质量，辅助运营和产品判断活跃盘子是否健康，并为促活、召回、互动链路优化提供数据依据。

核心目标如下：

1. 展示 DAU、活跃率、有效互动用户、有效互动率、沉默用户等核心指标。
2. 展示活跃趋势变化，帮助识别活跃规模和互动质量是否同步变化。
3. 展示活跃分层结构，帮助判断当前用户池是否存在沉默用户偏高或中高活跃不足的问题。
4. 提供按时间、终端查看活跃明细的能力，支持日常复盘和数据核对。

## 三、活跃统计

### 3.1 页面定位

活跃统计页位于统计管理模块下，用于从规模、质量、结构三个维度分析平台用户活跃情况。

页面不只展示活跃人数，而是强调：

1. 活跃规模是否稳定。
2. 活跃用户是否形成有效互动。
3. 活跃用户结构是否健康。
4. 沉默用户是否偏高。

### 3.2 页面结构

页面采用三排布局，筛选区域不计入三排结构。

第一部分：顶部筛选区  
第二部分：第一排统计卡片  
第三部分：第二排核心可视化区域  
第四部分：第三排活跃统计明细表

### 3.3 顶部筛选区

顶部筛选区用于统一控制统计口径时间范围。

展示内容：

1. 时间快捷筛选：今日、昨日、近 7 日、近 30 天
2. 日期范围选择器
3. 查询按钮
4. 重置按钮

产品要求：

1. 默认进入页面时，选中近 7 日。
2. 切换快捷时间后，日期选择器同步变化。
3. 点击查询后，刷新统计卡片、图表和明细表。
4. 点击重置后恢复默认近 7 日。

### 3.4 第一排统计卡片

第一排展示 5 张统计卡片：

1. DAU
2. 活跃率
3. 有效互动用户
4. 有效互动率
5. 沉默用户

卡片设计要求：

1. 每张卡片顶部展示指标名称。
2. 中部展示核心值。
3. 底部展示“判断型摘要信息”，不只展示定义说明。
4. 摘要信息优先体现“较前日变化”“当前占比”“分母口径”。

#### 3.4.1 DAU

含义：

- 当前统计周期内的活跃用户规模

展示要求：

1. 主值展示当前结果值。
2. 摘要信息展示“较前日变化 + 近 7 日日均”。

示例：

- 较前日 +1,150 人，7 日日均 10,851

#### 3.4.2 活跃率

含义：

- 活跃用户占可运营用户的比例

展示要求：

1. 主值展示百分比。
2. 摘要信息展示“较前日变化 + 当前分母规模”。

示例：

- 较前日 +0.8pp，分母 130,920

#### 3.4.3 有效互动用户

含义：

- 活跃用户中发生有效互动的去重人数

展示要求：

1. 主值展示人数。
2. 摘要信息展示“较前日变化 + 占 DAU 比例”。

示例：

- 较前日 +889 人，占 DAU 61.4%

#### 3.4.4 有效互动率

含义：

- 有效互动用户占活跃用户的比例

展示要求：

1. 主值展示百分比。
2. 摘要信息展示“较前日变化 + 分子/分母关系”。

示例：

- 较前日 +1.8pp，互动 6,952 / 活跃 11,323

#### 3.4.5 沉默用户

含义：

- 当前可运营用户池中的沉默用户规模

展示要求：

1. 主值展示人数。
2. 摘要信息展示“沉默占比 + 较前日变化 + 高活跃日均”。

示例：

- 沉默占比 91.4%，较前日 -0.7pp，高活跃日均 2,186 人

### 3.5 第二排核心可视化区域

第二排由左侧主图和右侧副图组成。

#### 3.5.1 左侧主图：活跃趋势

图表类型：

- 双柱 + 双折线组合图

展示指标：

1. DAU：柱状
2. 有效互动用户：柱状
3. 活跃率：折线
4. 有效互动率：虚线

设计目标：

1. DAU 与有效互动用户作为主规模指标，直观对比规模变化。
2. 活跃率与有效互动率作为质量指标，辅助判断活跃是否健康。
3. 图中需形成主次层次，避免四个指标同权竞争注意力。

交互要求：

1. hover 时展示当前日期四项指标完整数值。
2. 图例支持显示与隐藏。
3. 横轴按日期展示。
4. 左轴展示人数，右轴展示百分比。

图表文案要求：

标题：活跃趋势  
说明：跟踪 DAU、有效互动用户、活跃率与有效互动率的周期变化

#### 3.5.2 活跃趋势自动结论

主图底部需要展示自动结论，不只展示单一峰值描述。

建议至少包含以下信息：

1. 近 3 日活跃规模趋势：回升 / 回落 / 持平
2. 近 3 日有效互动率变化
3. 峰值日期与峰值 DAU

示例：

- 近 3 日活跃规模回升，有效互动率 +1.4pp；峰值日 2026-05-28，DAU 12,884

#### 3.5.3 右侧副图：活跃分层

图表类型：

- 环形图

当前展示分层：

1. 高活跃
2. 中活跃
3. 低活跃
4. 沉默用户

当前展示结构占比：

1. 高活跃：24%
2. 中活跃：31%
3. 低活跃：18%
4. 沉默用户：27%

当前版本产品要求：

1. 本期只展示分层结果，不定义规则条件。
2. 后续分层判定条件由产品另行补充。
3. 图表中心和底部文案应突出结构结论，而不是只展示图形。

建议结论：

- 中高活跃占比 55%，沉默用户占比 27%

### 3.6 第三排活跃统计明细表

明细表用于支持数据核对与日常复盘。

#### 3.6.1 表格字段

表格字段如下：

1. 统计日期
2. 可运营用户数
3. 活跃用户数
4. 活跃率
5. 有效互动用户数
6. 有效互动率
7. 沉默用户数
8. 活跃分层
9. 主活跃终端

#### 3.6.2 表格筛选

表格区域支持独立筛选。

筛选项包括：

1. 日期范围
2. 终端类型

产品要求：

1. 表格默认继承顶部筛选时间范围。
2. 表格支持单独查询和重置。
3. 默认终端为全部终端。

#### 3.6.3 活跃率与有效互动率列

展示要求：

1. 采用进度条 + 百分比展示方式。
2. 活跃率和有效互动率颜色应保持区分。
3. 数值保留 1 位小数。

#### 3.6.4 活跃分层列

活跃分层列用于在表格中展示用户结构，不适合采用多个标签并排方式。

当前展示方式：

1. 使用迷你堆叠进度条展示四层结构。
2. 通过颜色区分高活跃、中活跃、低活跃、沉默用户。
3. 条形下方展示简短主导层摘要，例如：
   - 中活跃主导 | 31.0%

hover tooltip 展示明细：

1. 分层名称
2. 对应人数
3. 对应占比

展示顺序固定：

1. 高活跃
2. 中活跃
3. 低活跃
4. 沉默用户

目标：

1. 在不增加表格拥挤度的前提下展示结构信息。
2. 一眼识别主导层。
3. hover 后可查看完整明细。

#### 3.6.5 主活跃终端列

当前版本展示主活跃终端。

展示要求：

1. 以标签形式展示 Android / iOS
2. 标签颜色应能快速区分不同终端
3. 当前版本不展示终端占比细分

### 3.7 指标口径定义

#### 3.7.1 可运营用户数

可运营用户数定义为满足以下条件的用户数：

1. 已注册
2. 未注销
3. 未删除
4. 未封禁
5. 可正常登录或使用核心功能

#### 3.7.2 活跃用户

活跃用户定义为统计周期内满足任一活跃行为的去重用户数。

当前版本建议按以下行为口径统计：

1. 成功登录
2. 进入核心业务页面
3. 产生消息、加好友等核心行为

#### 3.7.3 活跃率

活跃率计算公式：

`活跃率 = 活跃用户数 / 可运营用户数 × 100%`

说明：

1. 页面中的活跃率均使用该口径。
2. 建议通过 tooltip 对外说明分母口径。

#### 3.7.4 有效互动用户

有效互动用户定义为统计周期内满足以下任一行为的去重用户数：

1. 发送消息
2. 发起加好友

说明：

1. 当前版本先按“发消息或加好友”统计。
2. 后续可根据业务需要扩展行为范围。

#### 3.7.5 有效互动率

有效互动率计算公式：

`有效互动率 = 有效互动用户数 / 活跃用户数 × 100%`

说明：

1. 用于衡量活跃用户中有多少形成真实互动。
2. 不用于衡量新增用户激活质量。

#### 3.7.6 沉默用户

沉默用户定义为当前统计口径下未达到活跃标准的用户。

### 3.8 使用要求

#### 3.8.1 页面使用目标

页面主要面向以下角色：

1. 运营：判断是否需要促活或召回
2. 产品：观察活跃质量是否改善
3. 数据分析：核对活跃数据与结构变化
4. 管理层：快速查看活跃规模、质量和结构

#### 3.8.2 默认阅读路径

页面应支持以下阅读顺序：

1. 先看统计卡片判断当前结果
2. 再看活跃趋势判断近期变化
3. 再看活跃分层判断结构健康度
4. 最后通过明细表核对具体日期数据

#### 3.8.3 口径说明

以下指标建议在页面上提供说明入口：

1. 活跃率
2. 有效互动率
3. 沉默用户

示例说明：

- 活跃率 = 活跃用户数 / 可运营用户数
- 有效互动率 = 有效互动用户数 / 活跃用户数

#### 3.8.4 后续联动建议

活跃统计页面后续可与以下页面联动：

1. 用户分群
2. 用户动作
3. 召回策略
4. 用户画像

## 四、验收标准

### 4.1 页面结构验收

1. 页面采用三排布局。
2. 第一排展示 5 张统计卡片。
3. 第二排左侧为活跃趋势，右侧为活跃分层。
4. 第三排展示活跃统计明细表。

### 4.2 指标口径验收

1. 活跃率按“活跃用户 / 可运营用户”计算。
2. 有效互动率按“有效互动用户 / 活跃用户”计算。
3. 有效互动用户按“发消息或加好友”口径统计。
4. 沉默用户按当前未活跃用户统计。

### 4.3 数据展示验收

1. 统计卡片摘要需体现判断信息，不只是定义说明。
2. 活跃趋势图需同时展示规模指标和质量指标。
3. 活跃分层图需展示四类结构结果。
4. 明细表活跃分层需展示为迷你堆叠进度条。

### 4.4 交互验收

1. 时间筛选可正常切换并刷新页面数据。
2. 图表 hover 可查看明细。
3. 表格筛选与分页可正常使用。
4. 活跃分层 tooltip 可展示人数与占比明细。

## 五、后续扩展建议

2. 增加渠道来源筛选

4. 增加趋势异常提示
