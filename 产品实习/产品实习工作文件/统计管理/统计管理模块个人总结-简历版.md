# 统计管理模块个人总结（简历版）

## 1. 模块范围确认

结合当前前端菜单重写逻辑与页面目录，统计管理正式范围包含 5 个页面：

1. 用户增长统计
2. 留存统计
3. 转化统计
4. 风险异常统计
5. 活跃统计

对应源码目录：

- `src/views/statistical-manage/user-growth-stats`
- `src/views/statistical-manage/retention-stats`
- `src/views/statistical-manage/conversion-stats`
- `src/views/statistical-manage/risk-exception-stats`
- `src/views/statistical-manage/active-stats`

补充说明：

- `src/utils/menu-rewrite.js` 中明确补齐了以上 5 个统计页面的前端菜单入口。
- `src/views/statistical-manage/exception-stats` 为旧版残留页面，不在当前统计管理正式菜单挂载范围内，且接口仍为占位实现，不建议作为核心成果写入简历。

## 2. 模块整体分析

从当前代码实现看，统计管理模块已完成一套相对完整的前端数据分析看板能力，特点是：

- 统一采用 Vue 3 `script setup` + Ant Design Vue + ECharts 图表封装实现。
- 页面结构基本统一为“顶部时间筛选 + 核心指标卡片 + 可视化图表 + 明细表格”。
- 复用了公共筛选组件、表格体系、图表组件和卡片样式，模块一致性较强。
- 已完成前端指标模型、趋势结论文案、筛选联动、分页表格和多图表切换。
- 当前大部分页面数据来源仍以本地构造数据为主，接口层多数还是占位状态，说明该阶段更偏前端方案落地、页面建模和交互验证。

## 3. 各页面可提炼成果

### 3.1 用户增长统计

页面目标：
围绕用户规模变化，展示总用户数、净增长、注销、Android 新增、iOS 新增等核心指标。

前端实现亮点：

- 支持顶部时间筛选与表格内二次筛选联动。
- 将增长趋势与“新增 vs 注销”做成双图联动，便于快速判断增长质量。
- 支持按 Android / iOS 视角切换，统一重算总量、净增长和注销口径。
- 对统计卡片补充“较上周期”变化说明，增强管理视角。

### 3.2 留存统计

页面目标：
围绕新增用户后续活跃情况，展示 D1 / D3 / D7 留存人数和留存结构。

前端实现亮点：

- 单独设计了留存页面的时间规则，限制注册时间必须落在可观察成熟窗口内。
- 支持整体趋势与平台趋势切换，并支持 D1 / D3 / D7 维度快速切换。
- 用趋势图 + 结构图 + 明细表三层方式表达留存，不只是静态指标展示。
- 将留存页面抽象为 cohort 视角处理，具备较强的数据分析产品思路。

### 3.3 转化统计

页面目标：
围绕新用户激活路径，展示注册、登录、加好友、好友通过、首聊等转化漏斗。

前端实现亮点：

- 抽象了完整的新用户激活漏斗模型，覆盖注册到首聊的核心链路。
- 支持整体转化趋势与分端趋势切换，并可切换登录、加好友、好友通过、首聊等指标。
- 在明细表中增加“最大流失环节”字段，帮助快速识别转化短板。
- 增加漏斗图、趋势图、分端对比等多种可视化方式，页面分析属性较强。

### 3.4 风险异常统计

页面目标：
围绕异常事件与风险对象，展示风险用户、黑名单 IP、风险 IP、风险设备、异常事件等指标。

前端实现亮点：

- 区分“异常事件数”和“风险对象数”两套统计口径，表达较清晰。
- 同时支持时间筛选、终端筛选、异常类型 Tab 筛选，页面交互复杂度较高。
- 明细表不仅展示异常类型、风险等级，还补充用户信息、账号状态、IP、设备、建议动作等风控处置字段。
- 用趋势图和异常类型占比图辅助定位峰值日期和主要风险来源。

### 3.5 活跃统计

页面目标：
围绕 DAU、活跃率、有效互动、沉默用户等指标，反映用户活跃质量与分层情况。

前端实现亮点：

- 抽象了活跃用户、有效互动用户、沉默用户、高中低活跃分层等指标体系。
- 将活跃趋势和活跃分层结合展示，兼顾规模变化与结构变化。
- 明细表支持按时间与终端筛选，并展示主活跃终端和活跃分层结果。
- 指标卡片不仅展示主值，还补充日环比、7 日日均等辅助分析信息。
