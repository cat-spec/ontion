# IM后台留存统计需求文档

## 一、留存统计

### 1.1 文档说明

本文档用于说明 IM 后台「统计管理 / 留存统计」页面的产品需求，统一产品、前端、后端、测试、运营对页面目标、展示结构、统计口径、交互逻辑和验收标准的理解。

本文档基于当前项目已实现页面和最终确认页面效果整理，适用于当前前端实现版本。

当前版本说明如下：

1. 页面已在统计管理模块下独立实现为「留存统计」页面。
2. 页面数据当前为静态假数据，用于前端展示和交互验证。
3. 后续接入真实接口时，应保持本文档定义的页面结构和统计口径不变。
4. 本文档不包含接口设计、数据库设计、埋点设计和 BI SQL 实现细节。

### 1.2 模块定位

留存统计页面用于回答“新增用户是否在后续关键节点继续活跃”这一问题，帮助产品、运营和管理人员评估新增质量，而不是评估转化或增长规模。

页面核心解决以下问题：

| 问题 | 对应展示 |
| --- | --- |
| 当前统计周期内新增规模有多大 | 顶部统计卡片：新增用户数 |
| 新增用户在 D1 / D3 / D7 是否继续活跃 | 顶部统计卡片：次日留存用户、3日留存用户、7日留存用户 |
| 留存表现是整体问题还是平台差异问题 | 留存趋势图、留存结构图 |
| Android 与 iOS 留存差异有多大 | 留存趋势图平台视图、留存结构双头柱状图 |
| 每日新增 cohort 的 D1 / D3 / D7 明细如何 | 留存明细表 |

页面不承担以下目标：

1. 不展示转化漏斗。
2. 不展示用户增长或净增长结论。
3. 不展示复杂经营分析结论。
4. 不展示留存改善建议或自动诊断策略。

### 1.3 适用角色

| 角色 | 使用目标 |
| --- | --- |
| 产品经理 | 判断新增质量是否健康，观察不同平台的留存差异 |
| 运营人员 | 评估拉新后续质量，定位哪一段留存下滑更明显 |
| 管理人员 | 快速查看当前周期新增用户沉淀情况 |
| 测试人员 | 验证筛选、图表、表格、统计口径是否符合需求 |
| 开发人员 | 对齐页面结构、字段含义和统计范围 |

### 1.4 页面结构

页面采用三段式布局，顶部筛选区不计入三段主结构。

| 区域 | 内容 | 说明 |
| --- | --- | --- |
| 顶部筛选区 | 注册时间、查询、重置 | 控制整页统计区间 |
| 第一排统计卡片 | 新增用户数、次日留存用户、3日留存用户、7日留存用户 | 展示核心汇总指标 |
| 第二排核心可视化区域 | 左侧留存趋势图、右侧留存结构图 | 展示整体趋势与平台结构 |
| 第三排明细表 | 工具按钮、留存明细表 | 展示按日 cohort 明细 |

页面整体布局风格需要与现有统计管理模块中的「用户增长统计」「转化统计」保持一致，保证视觉复用和使用习惯一致。

### 1.5 顶部筛选区

顶部筛选区位于页面最上方，仅包含以下三个控件：

1. 注册时间
2. 查询按钮
3. 重置按钮

#### 1.5.1 展示要求

| 控件 | 展示形式 | 说明 |
| --- | --- | --- |
| 注册时间 | 单日期选择器 | 只选择开始时间 |
| 查询 | 主按钮 | 刷新整页数据 |
| 重置 | 次按钮 | 恢复默认开始时间 |

#### 1.5.2 交互规则

1. 注册时间只允许选择开始日期，不提供结束日期选择器。
2. 页面默认开始时间为“当前可观察截止日往前 29 天”。
3. 当前可观察截止日定义为“今天往前第 7 天的注册日”。
4. 点击查询后，刷新顶部统计卡片、留存趋势图、留存结构图和留存明细表。
5. 点击重置后，恢复默认开始时间并刷新整页数据。
6. 若用户选择的开始时间晚于当前可观察截止日，则页面内部自动按当前可观察截止日进行统计。

#### 1.5.3 统计时间范围规则

当前版本页面采用“单开始时间 + 系统自动截止”的轻筛选方式。

实际统计区间规则如下：

1. 统计开始时间 = 用户选择的注册开始日期。
2. 统计结束时间 = 当前可观察截止日。
3. 当前可观察截止日 = 当前日期减 7 天。
4. 页面所有模块必须使用同一统计区间。

示例：

若当前日期为 2026-06-02，则：

1. 当前可观察截止日为 2026-05-26。
2. 若用户选择注册时间为 2026-04-30，则实际统计区间为 2026-04-30 至 2026-05-26。
3. 若用户未修改筛选，则默认统计区间为 2026-04-27 至 2026-05-26。

### 1.6 顶部统计卡片

顶部展示 4 张统计卡片：

1. 新增用户数
2. 次日留存用户
3. 3日留存用户
4. 7日留存用户

当前版本不展示流失率统计卡片。

#### 1.6.1 卡片公共规则

1. 卡片采用与统计管理其他页面一致的视觉组件。
2. 每张卡片展示标题、主数值、右上状态标签、辅助说明和图标。
3. 主数值采用千分位格式。
4. 卡片辅助说明必须体现指标口径，不只展示泛化文案。
5. 卡片数量变化后布局需自动适配，不应出现空列。

#### 1.6.2 新增用户数

展示内容：

1. 标题：新增用户数
2. 主值：当前统计区间内新增用户总数
3. 状态标签：日均新增人数
4. 辅助说明 1：统计周期内新增注册用户
5. 辅助说明 2：统计区间起止日期

#### 1.6.3 次日留存用户

展示内容：

1. 标题：次日留存用户
2. 主值：当前统计区间内所有 cohort 的 D1 留存用户数之和
3. 状态标签：表现稳定 / 需关注
4. 辅助说明 1：D1 留存率
5. 辅助说明 2：基于已完成 7 日观察的新增用户

#### 1.6.4 3日留存用户

展示内容：

1. 标题：3日留存用户
2. 主值：当前统计区间内所有 cohort 的 D3 留存用户数之和
3. 状态标签：表现稳定 / 需关注
4. 辅助说明 1：D3 留存率
5. 辅助说明 2：基于已完成 7 日观察的新增用户

#### 1.6.5 7日留存用户

展示内容：

1. 标题：7日留存用户
2. 主值：当前统计区间内所有 cohort 的 D7 留存用户数之和
3. 状态标签：核心指标 / 需关注
4. 辅助说明 1：D7 留存率
5. 辅助说明 2：与留存结构图使用同一批成熟 cohort

### 1.7 留存趋势图

留存趋势图位于第二排左侧，是页面主图。

#### 1.7.1 图表类型

图表类型为折线图，当前版本展示留存人数，不展示留存率。

#### 1.7.2 展示结构

图表顶部右侧提供 Tab 切换：

1. 整体
2. 平台

当选中「平台」时，额外展示第二层留存周期切换：

1. 1日
2. 3日
3. 7日

#### 1.7.3 整体视图

当切换为「整体」时，图表展示三条折线：

1. D1
2. D3
3. D7

横轴：

- 注册日期

纵轴：

- 留存人数

#### 1.7.4 平台视图

当切换为「平台」时，图表展示两条折线：

1. Android
2. iOS

当前展示的留存周期由二级切换控制。

示例：

1. 平台 + 1日：展示 Android D1 与 iOS D1
2. 平台 + 3日：展示 Android D3 与 iOS D3
3. 平台 + 7日：展示 Android D7 与 iOS D7

#### 1.7.5 图表文案

标题：

- 留存趋势图

说明文案：

- 统一基于成熟 cohort 查看 D1 / D3 / D7 留存趋势，或切到平台对比 Android 与 iOS

#### 1.7.6 交互要求

1. Hover 时展示当前注册日期下各系列数值。
2. 图例需展示当前系列名称。
3. 整体 / 平台切换后，图表数据实时切换。
4. 平台模式下切换 1日 / 3日 / 7日 时，图表数据实时切换。
5. 图表使用当前顶部筛选后的统计区间。

### 1.8 留存结构

留存结构位于第二排右侧，用于展示平台维度下的留存与流失对比。

#### 1.8.1 图表类型

图表类型为双头柱状图。

当前设计规则如下：

1. 上半部分展示留存人数。
2. 下半部分展示流失人数。
3. 横轴展示 Android 与 iOS 两个平台。
4. 纵轴正方向展示留存，负方向展示流失。

#### 1.8.2 顶部切换

留存结构右上角提供留存周期切换：

1. 1日
2. 3日
3. 7日

默认选中：

- 7日

#### 1.8.3 展示内容

对每个平台，展示以下两部分：

1. 留存人数
2. 流失人数

其中：

1. 留存人数 = 当前统计区间内该平台注册 cohort 的对应留存人数之和
2. 流失人数 = 当前统计区间内该平台注册 cohort 的新增人数之和减去对应留存人数之和

#### 1.8.4 图表文案

标题：

- 留存结构

说明文案：

- 对比 Android 与 iOS 在统一 cohort 口径下的留存与流失结构

底部自动结论：

1. 输出当前区间内留存率更高的平台
2. 输出与另一平台的留存率差值
3. 输出当前所选留存周期

示例：

- 近 30 日 iOS 7日留存率 54.6%，较 Android 高 3.5pct；两端留存均高于流失。

#### 1.8.5 交互要求

1. Hover 时展示 Android / iOS 在当前留存周期下的留存与流失人数。
2. 切换 1日 / 3日 / 7日 后，柱体数据和底部结论同步刷新。
3. 图表使用与顶部统计卡片相同的 cohort 区间。

   - D3

### 1.10 统计口径

#### 1.10.1 总体口径

页面统一按“注册日 cohort 留存”计算，不按活跃日倒推统计。

统一规则如下：

1. 统计对象：去重用户。
2. 统计粒度：自然日。
3. cohort 基准：注册日期。
4. 平台归属：注册平台。
5. 活跃判定：当前版本为静态演示数据，后续真实数据接入时需统一定义有效活跃事件。
6. 同一用户同一天多次活跃只计 1 个留存用户。

#### 1.10.2 新增用户数

定义：

- 在某注册日期内完成注册的去重用户数

公式：

- 新增用户数(T) = count(distinct user_id where register_date = T)

#### 1.10.3 D1 留存用户

定义：

- T 日新增用户中，在 T+1 再次活跃的去重用户数

公式：

- D1留存用户(T) = count(distinct user_id in cohort(T) and active_date = T+1)

留存率：

- D1留存率(T) = D1留存用户(T) / 新增用户数(T)

#### 1.10.4 D3 留存用户

定义：

- T 日新增用户中，在 T+3 再次活跃的去重用户数

公式：

- D3留存用户(T) = count(distinct user_id in cohort(T) and active_date = T+3)

留存率：

- D3留存率(T) = D3留存用户(T) / 新增用户数(T)

#### 1.10.5 D7 留存用户

定义：

- T 日新增用户中，在 T+7 再次活跃的去重用户数

公式：

- D7留存用户(T) = count(distinct user_id in cohort(T) and active_date = T+7)

留存率：

- D7留存率(T) = D7留存用户(T) / 新增用户数(T)

#### 1.10.6 流失率

当前页面中的流失率固定定义为 7 日流失率。

公式：

- 流失率(T) = 1 - D7留存率(T)

#### 1.10.7 成熟 cohort 规则

页面所有模块必须只统计“已完成 D7 观察的新增 cohort”。

规则如下：

1. 若当前日期为 T，最新可统计注册日为 T-7。
2. T-6 至 T 的新增用户不能参与当前页面 D7 留存判断。
3. 顶部统计卡片、趋势图、结构图、明细表必须使用同一成熟 cohort 范围。

#### 1.10.8 平台口径

Android / iOS 对比统一按注册平台归属。

规则如下：

1. 用户在 Android 注册，则归属 Android cohort。
2. 用户在 iOS 注册，则归属 iOS cohort。
3. 用户后续跨平台登录，不改变其原始注册平台归属。

### 1.12 验收标准

#### 1.12.1 页面结构验收

1. 给定用户进入统计管理模块，当进入留存统计页面，则页面展示顶部筛选区、4 张统计卡片、2 个图表区域和 1 个明细表区域。
2. 给定页面已加载完成，当观察顶部统计卡片时，则仅展示新增用户数、次日留存用户、3日留存用户、7日留存用户 4 张卡片。
3. 给定页面处于桌面端分辨率，当观察页面布局时，则左侧为留存趋势图，右侧为留存结构图，下方为留存明细表。

#### 1.12.2 顶部筛选验收

1. 给定页面默认加载，当查看注册时间筛选时，则只展示单个开始时间选择器，不展示结束时间选择器。
2. 给定用户选择新的注册开始时间并点击查询，当页面刷新后，则统计卡片、图表和表格均按新时间范围刷新。
3. 给定用户点击重置，当页面刷新后，则注册开始时间恢复默认值，整页数据恢复默认区间。

#### 1.12.3 趋势图验收

1. 给定趋势图处于整体视图，当查看图例时，则展示 D1、D3、D7 三条折线。
2. 给定趋势图切换为平台视图，当查看图例时，则展示 Android 与 iOS 两条折线。
3. 给定趋势图已切换到平台视图，当切换 1日 / 3日 / 7日 时，则折线数据同步变化。

#### 1.12.4 留存结构验收

1. 给定留存结构图默认加载，当查看图形时，则展示 Android 与 iOS 两个平台的双头柱状图。
2. 给定切换 1日 / 3日 / 7日 时，当图表刷新后，则留存柱和流失柱同步变化。
3. 给定图表底部结论存在，当切换周期后，则结论文案也同步变化。

#### 1.12.5 明细表验收

1. 给定留存明细表加载完成，当查看字段时，则依次展示日期、新增、D1、D3、D7、流失率。
2. 给定查看 D1 / D3 / D7 任一列时，则同一单元格内第一行展示人数，第二行展示留存率。
3. 给定查看流失率列时，则同时展示百分比和进度条。
4. 给定点击刷新按钮时，则明细表数据重新加载。

#### 1.12.6 统计口径验收

1. 给定当前日期为 2026-06-02，当页面统计 D7 相关指标时，则最晚只统计到 2026-05-26 的注册 cohort。
2. 给定顶部卡片与图表同时存在，当核对统计区间时，则卡片、趋势图、结构图、明细表必须使用同一注册 cohort 区间。
3. 给定 Android / iOS 平台对比，当核对平台归属时，则统一按注册平台归属。

### 1.13 后续扩展建议

当前版本已满足前端静态展示和页面结构确认需求，后续如进入真实数据接入阶段，建议按以下方向扩展：

1. 增加真实活跃事件口径定义。
2. 增加快捷时间选项，如近 7 日、近 15 日、近 30 日。
3. 增加留存率与留存人数双维切换。
4. 增加异常 cohort 快捷筛选能力。
5. 增加导出真实数据能力。
