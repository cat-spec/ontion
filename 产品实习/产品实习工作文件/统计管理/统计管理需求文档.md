# 用户增长统计

## 1. 模块概述

## 2. 页面结构
  
页面采用三层结构：

### 2.1 顶部筛选区
用于选择全页统计周期。

### 2.2 顶部统计区

用于展示当前筛选周期内的核心汇总指标。

### 2.3 中部核心图表区

用于展示增长趋势与新增/注销关系。


### 2.4 底部明细区

用于展示按日拆分的用户增长明细及辅助筛选。


## 3. 顶部筛选区

### 3.1 功能说明

顶部筛选区作为全页统一筛选入口，控制顶部统计区与中部图表区的数据范围。


### 3.2 筛选项

1. 时间快捷筛选

- 今日

- 昨日

- 近7日

- 近30天


2. 自定义日期区间

- 开始日期

- 结束日期
  

3. 操作按钮

- 查询

- 重置


### 3.3 交互规则

1. 点击时间快捷项后，自动回填日期区间。

2. 手动选择日期区间后，取消快捷时间高亮状态。

3. 点击“查询”后，刷新顶部统计和中部图表。

4. 点击“重置”后，恢复默认值。

5. 默认时间范围为近7日。

### 3.4 页面联动范围

顶部筛选区联动：

- 顶部统计区

- 用户增长趋势图

- 新增 vs 注销图


## 4. 顶部统计区

### 4.1 功能说明

用于展示当前统计周期内的核心增长汇总结果。

### 4.2 展示指标

1. 用户总数

- 含义：截止统计结束日系统累计有效用户总数

2. 净增长用户数

- 含义：统计周期内新增用户数减去注销用户数


3. 注销用户数

- 含义：统计周期内完成注销或流失出库的用户数

4. Android 新增用户

- 含义：统计周期内 Android 端注册新增用户数


6. iOS 新增用户

- 含义：统计周期内 iOS 端注册新增用户数


### 4.3 展示规则

1. 每张统计卡展示：

- 指标标题

- 指标数值

- 指标说明文案

- 辅助图标

2. 数值使用千分位格式展示。

3. 统计卡采用统一白色卡片样式，保持与系统首页卡片风格一致。

  
### 4.4 优先级说明

顶部统计区必须包含“新增用户数”和“注销用户数”，因为两者共同构成净增长判断基础。

  

## 5. 用户增长趋势

### 5.1 功能说明

用于按天展示用户增长变化趋势，帮助判断周期内增长波动与平台贡献变化。

### 5.2 图表类型

双 Y 轴组合图
  

### 5.3 展示指标

1. 用户总量

- 图形类型：折线图

- 轴：右侧 Y 轴

- 含义：每日截止时点累计有效用户数
  

2. Android 新增

- 图形类型：柱状图

- 轴：左侧 Y 轴

- 含义：每日 Android 端新增注册用户数

  

3. iOS 新增

- 图形类型：柱状图

- 轴：左侧 Y 轴

- 含义：每日 iOS 端新增注册用户数

  

4. 净增长用户数

- 图形类型：折线图

- 轴：左侧 Y 轴

- 含义：每日新增减去每日注销后的净增长人数


### 5.4 设计目的

该图重点回答：

- 用户总规模是否稳定上升

- 哪个平台在拉动新增

- 净增长是否持续健康

  

### 5.5 展示规则

1. 横轴按天展示。

2. 左侧 Y 轴展示新增/净增长人数。

3. 右侧 Y 轴展示用户总量。

4. 图例需支持展示：

- 用户总量

- Android 新增

- iOS 新增

- 净增长用户数

  

### 5.6 交互要求

1. Hover 展示当日完整数据：

- 日期

- 用户总量

- Android 新增

- iOS 新增

- 净增长用户数

  
2. 图例点击可控制系列显隐。

## 6. 新增 vs 注销

### 6.1 功能说明

用于展示统计周期内每日新增用户与注销用户的对比关系，帮助判断增长质量。

### 6.2 图表类型

正负柱状图

  

### 6.3 展示指标

1. 新增用户

- 正向柱

- 含义：每日新增注册用户数

  

2. 注销用户

- 负向柱

- 含义：每日注销用户数

  

### 6.4 展示目的

该图重点回答：

- 新增是否被注销抵消

- 哪些日期增长表现最好

- 哪些日期注销压力较高

  

### 6.5 摘要说明

图表标题下方需展示一条自动摘要，示例：

- 近7日平均净增长 431，峰值出现在 2026-05-29，净增长 483

  

### 6.6 交互要求

1. Hover 时展示：

- 日期

- 新增用户数

- 注销用户数

- 当日净增长用户数

  

## 7. 用户增长明细

  

### 7.1 功能说明

用于以表格方式展示按日拆分的增长数据，并支持局部筛选与排序。

  

### 7.2 区域结构

1. 标题

- 用户增长明细

  

2. 筛选工具

- 日期范围

- 渠道

- 查询

- 重置

  

3. 数据表格

  

### 7.3 筛选项说明

1. 日期范围

- 作用范围：仅作用于明细表

  

2. 渠道

- Android

- iOS

- 不选择时视为全部

  

### 7.4 表格字段

1. 日期

2. 总用户数

3. 新增用户数

4. 净增长用户数

5. Android 新增

6. iOS 新增

7. 注销用户数

8. 新增主力平台

  

### 7.5 字段口径

1. 总用户数

- 截止当日累计有效用户数

  

2. 新增用户数

- 当日首次注册成功用户数

  

3. 净增长用户数

- 当日新增用户数减去当日注销用户数

  

4. Android 新增

- 当日 Android 端新增注册用户数

  

5. iOS 新增

- 当日 iOS 端新增注册用户数

  

6. 注销用户数

- 当日注销用户数

  

7. 新增主力平台

- 当日新增更多的平台

- 展示值：Android / iOS / 持平


### 7.6 排序要求

表格需支持以下关键列排序：

- 日期

- 新增用户数

- 净增长用户数

- 注销用户数

### 7.7 使用目的

底部表格用于支撑进一步核查，帮助用户定位：

- 哪天新增最高

- 哪天净增长最低

- 哪天注销最多

- 哪个平台在每日层面贡献更高


## 8. 数据口径与业务规则

  
### 8.1 用户总数

指截至统计结束时间仍为有效状态的累计用户。

  


### 8.2 注销用户

指统计周期内完成注销或被认定流失出库的用户。

  

### 8.3 净增长用户

计算公式：

净增长用户数 = 新增用户数 - 注销用户数

  

### 8.4 平台口径

平台按用户注册来源设备区分：

- Android

- iOS

### 8.5 新增主力平台判定规则

1. Android 新增 > iOS 新增，展示 Android

2. iOS 新增 > Android 新增，展示 iOS

3. Android 新增 = iOS 新增，展示 持平

# 转化统计

## 一、模块定位


核心分析链路如下：

```text
注册用户 → 登录用户 → 发起加好友用户 → 好友通过用户 → 首聊用户
```

页面核心需要回答以下问题：

| 问题                    | 对应展示                 |
| --------------------- | -------------------- |
| 当前周期新用户规模是多少          | 顶部统计卡片：新增注册用户        |
| 新注册用户中有多少完成登录         | 顶部统计卡片：新增登录用户数、登录转化率 |
| 用户是否愿意建立关系            | 顶部统计卡片：加好友转化率        |
| 用户是否完成 IM 核心激活        | 顶部统计卡片：全链路首聊转化率      |
| 用户主要卡在哪一步             | 新用户激活漏斗、最大流失环节       |
| Android 与 iOS 哪个端转化更弱 | 终端转化对比               |
| 每日、每端的明细数据如何          | 转化统计明细表              |

## 二、页面结构

页面由四个区域组成：

| 区域     | 内容                                   |
| ------ | ------------------------------------ |
| 顶部筛选区  | 时间筛选，支持今日、昨日、近 7 日、近 30 天、自定义日期范围    |
| 顶部统计卡片 | 新增注册用户、新增登录用户数、登录转化率、加好友转化率、全链路首聊转化率 |
| 核心可视化区 | 新用户激活漏斗、转化率趋势、终端转化对比                 |
| 明细列表区  | 日期范围、终端类型筛选、转化统计明细表                  |

页面默认统计近 7 日数据。顶部筛选区控制顶部卡片和三张核心图表。明细列表区默认同步顶部日期范围，并支持单独按日期范围和终端类型查询。
## 三、顶部筛选区

顶部筛选区复用用户增长统计页面的时间筛选组件。

| 筛选项     | 说明                      |
| ------- | ----------------------- |
| 今日      | 统计当天新注册用户的转化表现          |
| 昨日      | 统计昨日新注册用户的转化表现          |
| 近 7 日   | 默认选项，统计最近 7 日新注册用户的转化表现 |
| 近 30 天  | 统计最近 30 天新注册用户的转化表现     |
| 自定义日期范围 | 用户手动选择开始日期和结束日期         |
| 查询      | 按当前日期范围刷新顶部卡片和核心图表      |
| 重置      | 恢复默认近 7 日               |

日期口径按注册日期归属。选择某个日期范围后，统计该范围内新增注册用户后续是否完成登录、发起加好友、好友通过和首聊。

## 四、顶部统计卡片

顶部展示 5 个统计卡片。卡片主数值展示核心指标，底部展示辅助数据。

| 卡片       | 主指标              | 辅助展示                          | 说明                |
| -------- | ---------------- | ----------------------------- | ----------------- |
| 新增注册用户   | 新增注册用户数          | Android 新增注册用户数 / iOS 新增注册用户数 | 展示统计周期内完成注册的新用户规模 |
| 新增登录用户数  | 新增登录用户数          | Android 登录用户数 / iOS 登录用户数     | 展示新注册用户中完成登录的人数   |
| 登录转化率    | 登录用户数 / 新增注册用户数  | 登录用户数 / 新增注册用户数               | 判断注册后进入产品的比例      |
| 加好友转化率   | 发起加好友用户数 / 登录用户数 | 发起加好友用户数 / 登录用户数              | 判断登录用户是否产生建立关系意愿  |
| 全链路首聊转化率 | 首聊用户数 / 新增注册用户数  | 首聊用户数 / 新增注册用户数               | 判断新用户是否完成 IM 核心激活 |
## 五、核心可视化区

核心可视化区一排展示三张卡片。

### 5.1 新用户激活漏斗

新用户激活漏斗使用 ECharts 漏斗图展示。

漏斗节点如下：

```text
注册用户 → 登录用户 → 发起加好友用户 → 好友通过用户 → 首聊用户
```

图内展示：

| 展示项  | 说明                            |
| ---- | ----------------------------- |
| 节点名称 | 注册用户、登录用户、发起加好友用户、好友通过用户、首聊用户 |
| 节点人数 | 当前节点命中的用户数量                   |

鼠标悬停 tooltip 展示：

| 节点      | tooltip 展示       |
| ------- | ---------------- |
| 注册用户    | 用户数、注册基数         |
| 登录用户    | 用户数、登录转化率        |
| 发起加好友用户 | 用户数、加好友转化率       |
| 好友通过用户  | 用户数、好友通过率        |
| 首聊用户    | 用户数、好友通过 → 首聊转化率 |

tooltip 不展示总转化率。

卡片底部展示结论：

```text
最大流失环节：登录 → 加好友，流失 1,552 人
```

最大流失环节计算方式：

| 环节         | 流失人数               |
| ---------- | ------------------ |
| 注册 → 登录    | 新增注册用户数 - 登录用户数    |
| 登录 → 加好友   | 登录用户数 - 发起加好友用户数   |
| 加好友 → 好友通过 | 发起加好友用户数 - 好友通过用户数 |
| 好友通过 → 首聊  | 好友通过用户数 - 首聊用户数    |

流失人数最大的环节即为最大流失环节。

### 5.2 转化率趋势

转化率趋势使用折线图展示统计周期内每日转化率变化。

趋势线如下：

| 趋势线      | 计算方式               |
| -------- | ------------------ |
| 登录转化率    | 登录用户数 / 新增注册用户数    |
| 加好友率     | 发起加好友用户数 / 登录用户数   |
| 好友通过率    | 好友通过用户数 / 发起加好友用户数 |
| 全链路首聊转化率 | 首聊用户数 / 新增注册用户数    |

横轴为日期，纵轴为百分比。图例展示四条转化率曲线。

卡片底部展示结论：

```text
全链路首聊转化率最低：2026-05-26，15.4%
```

该结论用于快速定位统计周期内首聊转化表现最弱的日期。

### 5.3 终端转化对比

终端转化对比使用柱状图展示 Android 与 iOS 两个终端在关键激活环节的转化质量。

对比指标如下：

| 指标       | 计算方式               |
| -------- | ------------------ |
| 登录转化率    | 登录用户数 / 新增注册用户数    |
| 加好友转化率   | 发起加好友用户数 / 登录用户数   |
| 好友通过率    | 好友通过用户数 / 发起加好友用户数 |
| 全链路首聊转化率 | 首聊用户数 / 新增注册用户数    |

卡片底部展示结论：

```text
iOS 全链路首聊转化率偏低，终端差值 0.2%
```

该结论用于快速识别首聊转化表现较弱的终端。

## 六、转化统计明细

明细列表用于按日期和终端查看转化数据。列表支持分页和核心字段排序。

### 6.1 列表筛选

| 筛选项  | 说明                  |
| ---- | ------------------- |
| 日期范围 | 查询明细数据的统计日期范围       |
| 终端类型 | 支持 Android、iOS，允许清空 |
| 查询   | 按日期范围和终端类型刷新列表      |
| 重置   | 恢复当前顶部日期范围，并清空终端类型  |
### 6.2 列表字段

| 字段       | 说明                      |
| -------- | ----------------------- |
| 统计日期     | 数据所属日期                  |
| 终端类型     | Android 或 iOS           |
| 新增注册用户数  | 当日该终端新增注册用户数量           |
| 登录用户数    | 当日该终端新增注册用户中完成登录的人数     |
| 登录转化率    | 登录用户数 / 新增注册用户数         |
| 发起加好友用户数 | 当日该终端新增注册用户中发起加好友的人数    |
| 加好友转化率   | 发起加好友用户数 / 登录用户数        |
| 好友通过用户数  | 当日该终端新增注册用户中成功建立好友关系的人数 |
| 好友通过率    | 好友通过用户数 / 发起加好友用户数      |
| 首聊用户数    | 当日该终端新增注册用户中发送首条消息的人数   |
| 全链路首聊转化率 | 首聊用户数 / 新增注册用户数         |
| 最大流失环节   | 当日该终端流失人数最多的转化环节        |

### 6.3 排序规则

列表只对核心字段提供排序，不要求所有字段都支持排序。

支持排序字段如下：

| 字段       | 排序规则      |
| -------- | --------- |
| 统计日期     | 按日期升序或降序  |
| 新增注册用户数  | 按数值升序或降序  |
| 登录用户数    | 按数值升序或降序  |
| 登录转化率    | 按百分比升序或降序 |
| 发起加好友用户数 | 按数值升序或降序  |
| 加好友转化率   | 按百分比升序或降序 |
| 好友通过用户数  | 按数值升序或降序  |
| 好友通过率    | 按百分比升序或降序 |
| 首聊用户数    | 按数值升序或降序  |
| 全链路首聊转化率 | 按百分比升序或降序 |

终端类型和最大流失环节不要求排序。

## 七、数据口径

| 指标       | 口径                      |
| -------- | ----------------------- |
| 新增注册用户数  | 统计周期内完成注册的新用户数          |
| 新增登录用户数  | 新增注册用户中至少成功登录一次的用户数     |
| 发起加好友用户数 | 新增注册用户中至少发起一次加好友申请的用户数  |
| 好友通过用户数  | 新增注册用户中至少成功建立一个好友关系的用户数 |
| 首聊用户数    | 新增注册用户中至少发送一条单聊消息的用户数   |
| 登录转化率    | 新增登录用户数 / 新增注册用户数       |
| 加好友转化率   | 发起加好友用户数 / 新增登录用户数      |
| 好友通过率    | 好友通过用户数 / 发起加好友用户数      |
| 全链路首聊转化率 | 首聊用户数 / 新增注册用户数         |

计算百分比时保留 1 位小数。

如果分母为 0，转化率按 0% 展示。

## 八、交互规则

1. 页面首次进入时默认选中近 7 日。
2. 顶部时间筛选变化后，顶部统计卡片、激活漏斗、转化率趋势、终端转化对比同步刷新。
3. 明细列表默认使用顶部时间范围。
4. 明细列表点击查询后，按列表日期范围和终端类型刷新。

### 1.4 异常与风险口径说明

风险异常统计页面中，「异常」和「风险」属于不同统计口径。

异常是事件口径，描述系统捕获到多少次可疑行为或异常命中。异常事件允许同一用户、同一 IP、同一设备在同一统计周期内多次命中，因此异常事件数可以重复累计。

风险是对象口径，描述多少用户、IP 或设备因异常行为被判定为存在风险。风险对象通常按用户、IP 或设备去重统计。

| 类型      | 口径      | 是否允许重复 | 示例                          |
| ------- | ------- | ------ | --------------------------- |
| 异常事件数   | 事件口径    | 允许重复   | 同一用户一天命中 3 次屏蔽词，计 3 次异常事件   |
| 风险用户数   | 用户对象口径  | 去重     | 同一用户一天命中多次异常，风险用户数通常计 1 人   |
| 风险 IP 数 | IP 对象口径 | 去重     | 同一 IP 多次触发异常，风险 IP 数通常计 1 个 |
| 风险设备数   | 设备对象口径  | 去重     | 同一设备多次触发异常，风险设备数通常计 1 台     |

### 1.5 指标统计口径

| 指标       | 统计口径                         |
| -------- | ---------------------------- |
| 异常事件数    | 统计周期内所有异常类型命中次数之和，允许同一对象重复命中 |
| 风险用户数    | 统计周期内命中过异常并被归入风险池的去重用户数      |
| 高风险用户数   | 风险用户中风险等级为高的去重用户数            |
| 中风险用户数   | 风险用户中风险等级为中的去重用户数            |
| 低风险用户数   | 风险用户中风险等级为低的去重用户数            |
| 风险层级分布   | 高风险用户数、中风险用户数、低风险用户数组成的分布    |
| 风险 IP 数  | 统计周期内命中过异常规则的去重 IP 数         |
| 黑名单 IP 数 | 风险 IP 中已进入黑名单或强管控状态的 IP 数    |
| 风险设备数    | 统计周期内命中过异常规则的去重设备数           |
| 设备异常     | 设备维度异常命中次数                   |
| IP 异常    | IP 维度异常命中次数                  |
| 屏蔽词命中    | 消息内容命中屏蔽词或敏感词的次数             |
| 账号异常     | 登录、注册、账号状态等账号维度异常命中次数        |
| 高频加好友    | 短时间内频繁发起加好友行为的异常次数           |
| 高频群发     | 短时间内频繁群发或批量发送相似内容的异常次数       |
| 主要异常类型   | 当日或当前周期内异常事件数最高的异常类型         |

风险层级分布规则：

1. 风险层级分布按用户数统计。
2. 高风险用户数、中风险用户数、低风险用户数应合并展示为一个堆叠进度条。
3. 三类风险用户数相加应等于或接近风险用户数。
4. 鼠标悬停堆叠进度条时，应展示高中低风险用户数明细。

# 活跃统计

### 1 页面结构

页面采用三排布局，筛选区域不计入三排结构。

第一部分：顶部筛选区  
第二部分：第一排统计卡片

第四部分：第三排活跃统计明细表

### 1.1 顶部筛选区

顶部筛选区用于统一控制统计口径时间范围。

展示内容：

1. 时间快捷筛选：今日、昨日、近 7 日、近 30 天
2. 日期范围选择器
3. 查询按钮
4. 重置按钮

产品要求：

1. 默认进入页面时，选中近 7 日。
2. 切换快捷时间后，日期选择器同步变化。
3. 点击查询后，刷新统计卡片、图表和明细表。
4. 点击重置后恢复默认近 7 日。

### 1.2 第一排统计卡片

第一排展示 5 张统计卡片：

1. DAU
2. 活跃率
3. 有效互动用户
4. 有效互动率
5. 沉默用户

卡片设计要求：

1. 每张卡片顶部展示指标名称。
2. 中部展示核心值。
3. 底部展示“判断型摘要信息”，不只展示定义说明。
4. 摘要信息优先体现“较前日变化”“当前占比”“分母口径”。

#### 1.2.1 DAU

含义：

- 当前统计周期内的活跃用户规模

展示要求：

1. 主值展示当前结果值。
2. 摘要信息展示“较前日变化 + 近 7 日日均”。

示例：

- 较前日 +1,150 人，7 日日均 10,851

#### 1.2.2 活跃率

含义：

- 活跃用户占可运营用户的比例

展示要求：

1. 主值展示百分比。
2. 摘要信息展示“较前日变化 + 当前分母规模”。

示例：

- 较前日 +0.8，分母 130,920

#### 1.2.3 有效互动用户

含义：

- 活跃用户中发生有效互动的去重人数

展示要求：

1. 主值展示人数。
2. 摘要信息展示“较前日变化 + 占 DAU 比例”。

示例：

- 较前日 +889 人，占 DAU 61.4%

#### 1.2.4 有效互动率

含义：

- 有效互动用户占活跃用户的比例

展示要求：

1. 主值展示百分比。
2. 摘要信息展示“较前日变化 + 分子/分母关系”。

示例：

- 互动 6,952 / 活跃 11,323

#### 1.2.5 沉默用户

含义：

- 当前可运营用户池中的沉默用户规模

展示要求：

1. 主值展示人数。
2. 摘要信息展示“沉默占比 + 较前日变化 + 高活跃日均”。

示例：

- 沉默占比 91.4%

### 1.3 第二排核心可视化区域

第二排由左侧主图和右侧副图组成。

#### 1.3.1 左侧主图：活跃趋势

图表类型：

- 双柱 + 双折线组合图

展示指标：

1. DAU：柱状
2. 有效互动用户：柱状
3. 活跃率：折线
4. 有效互动率：虚线

设计目标：

1. DAU 与有效互动用户作为主规模指标，直观对比规模变化。
2. 活跃率与有效互动率作为质量指标，辅助判断活跃是否健康。
3. 图中需形成主次层次，避免四个指标同权竞争注意力。

交互要求：

1. hover 时展示当前日期四项指标完整数值。
2. 图例支持显示与隐藏。
3. 横轴按日期展示。
4. 左轴展示人数，右轴展示百分比。

图表文案要求：

标题：活跃趋势  
说明：跟踪 DAU、有效互动用户、活跃率与有效互动率的周期变化

#### 1.3.2 活跃趋势自动结论

主图底部需要展示自动结论，不只展示单一峰值描述。

### 3.7 指标口径定义

#### 3.7.1 可运营用户数

可运营用户数定义为满足以下条件的用户数：

1. 已注册
2. 未注销
3. 未删除
4. 未封禁
5. 可正常登录或使用核心功能

#### 3.7.2 活跃用户

活跃用户定义为统计周期内满足任一活跃行为的去重用户数。

当前版本建议按以下行为口径统计：

1. 成功登录
2. 进入核心业务页面
3. 产生消息、加好友等核心行为

#### 3.7.3 活跃率

活跃率计算公式：

`活跃率 = 活跃用户数 / 可运营用户数 × 100%`

说明：

1. 页面中的活跃率均使用该口径。
2. 建议通过 tooltip 对外说明分母口径。

#### 3.7.4 有效互动用户

有效互动用户定义为统计周期内满足以下任一行为的去重用户数：

1. 发送消息
2. 发起加好友

说明：

1. 当前版本先按“发消息或加好友”统计。
2. 后续可根据业务需要扩展行为范围。

#### 3.7.5 有效互动率

有效互动率计算公式：

`有效互动率 = 有效互动用户数 / 活跃用户数 × 100%`

说明：

1. 用于衡量活跃用户中有多少形成真实互动。
2. 不用于衡量新增用户激活质量。

#### 3.7.6 沉默用户

沉默用户定义为当前统计口径下未达到活跃标准的用户。

### 1.2 模块定位

页面核心解决以下问题：

### 1.10 统计口径

#### 1.10.1 总体口径

页面统一按“注册日 cohort 留存”计算，不按活跃日倒推统计。

统一规则如下：

1. 统计对象：去重用户。
2. 统计粒度：自然日。
3. cohort 基准：注册日期。
4. 平台归属：注册平台。
5. 活跃判定：当前版本为静态演示数据，后续真实数据接入时需统一定义有效活跃事件。
6. 同一用户同一天多次活跃只计 1 个留存用户。

#### 1.10.2 新增用户数

定义：

- 在某注册日期内完成注册的去重用户数

#### 1.10.3 D1 留存用户

定义：

- T 日新增用户中，在 T+1 再次活跃的去重用户数

留存率：

- D1留存率(T) = D1留存用户(T) / 新增用户数(T)

#### 1.10.4 D3 留存用户

定义：

- T 日新增用户中，在 T+3 再次活跃的去重用户数

留存率：

- D3留存率(T) = D3留存用户(T) / 新增用户数(T)

#### 1.10.5 D7 留存用户

定义：

- T 日新增用户中，在 T+7 再次活跃的去重用户数

留存率：

- D7留存率(T) = D7留存用户(T) / 新增用户数(T)

%% #### 1.10.6 流失率

当前页面中的流失率固定定义为 7 日流失率。

公式：

- 流失率(T) = 1 - D7留存率(T) %%

#### 1.10.7 成熟 cohort 规则

页面所有模块必须只统计“已完成 D7 观察的新增 cohort”。

规则如下：

1. 若当前日期为 T，最新可统计注册日为 T-7。
2. T-6 至 T 的新增用户不能参与当前页面 D7 留存判断。
3. 顶部统计卡片、趋势图、结构图、明细表必须使用同一成熟 cohort 范围。

#### 1.10.8 平台口径

Android / iOS 对比统一按注册平台归属。

规则如下：

1. 用户在 Android 注册，则归属 Android cohort。
2. 用户在 iOS 注册，则归属 iOS cohort。
3. 用户后续跨平台登录，不改变其原始注册平台归属。
