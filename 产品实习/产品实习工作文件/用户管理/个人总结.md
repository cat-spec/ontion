# 系统设计
在本项目中，我负责 IM 后台「用户管理」模块的完整产品设计工作，覆盖用户列表、用户详情、用户画像、用户标签、用户分群、用户动作、活跃度评分、风险等级判断及可视化统计口径等核心能力。

我首先梳理了后台用户管理的整体业务流程，将用户管理体系拆解为「用户数据管理 → 用户标签识别 → 用户分群圈选 → 用户动作执行 → 效果复盘」的完整闭环。通过该流程，系统不仅能够完成基础用户查询和账号管理，也能支持运营、客服、风控等角色基于用户状态开展精细化管理。

在用户基础管理方面，我设计了所有用户列表、筛选条件、统计卡片、用户新增、编辑、批量操作、导出、登录日志、账号状态管理等功能，明确了用户总量、今日新增、今日登录、在线人数、黑名单用户等关键指标口径。

在用户详情方面，我设计了用户完整档案页面，覆盖基础资料、账号状态、生命周期标签、活跃等级、风险等级、好友关系网、社交信息、行为趋势、关键指标、标签摘要、风险摘要、事件流和处理记录，帮助运营、客服、风控人员快速判断用户是否正常、是否值得运营、是否存在风险以及下一步应采取的动作。

在用户画像方面，我设计了整体用户结构分析能力，包括活跃用户、核心用户、风险用户、资料完整度等核心指标，以及生命周期分布、用户分层、近 7 日活跃趋势、关系洞察、风险信号和用户明细等图表，用于帮助团队从整体视角分析用户质量和风险变化。

在标签体系方面，我设计了系统自动标签和人工标签两类标签能力，并规划了生命周期标签、活跃度标签、价值标签、社交标签、资料标签、来源标签、风控标签、客服标签和运营标签等分类。通过标签体系，将用户状态标准化，为后续分群和运营动作提供统一依据。

在用户分群方面，我将分群定位为“标签组合器”，避免分群规则与标签功能重复。分群规则主要通过包含标签、排除标签、时间窗口、逻辑关系和优先级进行配置，支持静态分群和动态分群两种模式。静态分群用于人工维护名单，动态分群用于沉默召回、高风险观察、新用户激活、核心用户维护等场景。

在用户动作方面，我设计了用户动作作为运营、客服、风控动作方案库。动作支持消息触达、权益发放、人工任务、风控处理四类类型。新建动作时不直接选择分群，而是生成“未执行”的动作方案；真正执行时从用户分群页面发起，将未执行动作绑定到指定分群，并进入执行和复盘流程。

同时，我补充设计了活跃度评分和风险等级判断规则。活跃度评分从登录活跃、消息互动、社交关系、留存稳定、资料完整和风险扣分等维度进行综合评估；风险等级则从设备风险、登录风险、行为风险、投诉风险和资料风险等维度判断用户风险水平，为用户分层、分群和风控动作提供依据。

通过本次设计，用户管理模块从单纯的用户信息维护，升级为支持用户识别、用户分层、精准分群、运营触达、客服跟进、风控处理和效果复盘的一套完整用户运营管理体系。该设计提升了后台用户管理的业务闭环能力，也为后续数据化运营和精细化风控提供了产品基础。

# 精彩亮点

1. 搭建「用户 → 标签 → 分群 → 动作 → 复盘」的用户运营闭环，将传统用户管理从基础信息维护升级为支持精细化运营、客服跟进和风控处理的用户管理体系。
2. - 设计活跃度评分模型，基于登录活跃、消息互动、社交关系、留存稳定、资料完整度和风险扣分等维度评估用户活跃水平，为用户分层和运营决策提供量化依据。
3. 设计风险等级判断模型，基于设备风险、登录风险、行为风险、投诉风险和资料风险识别高、中、低风险用户，支撑风控观察、限制处理和异常用户复核等场景。
4. 设计用户动作模块，支持消息触达、权益发放、人工任务、风控处理四类动作，并形成“新建动作方案 → 分群执行动作 → 状态流转 → 效果复盘”的运营执行链路。
