# IM后台用户动作与分群预设需求文档

## 一、文档说明

本文档用于说明 IM 后台「用户动作」与「用户分群执行动作」的产品需求，重点定义预设动作、动作规则、分群预设、执行流程、页面展示和验收标准。

本文档不包含后端接口、数据库设计、代码实现、权限点等技术内容。文档面向产品、前端、测试、运营、客服、风控等角色，用于统一理解用户动作功能的业务目标和页面逻辑。

## 二、产品定位

用户动作用于承接用户分群之后的运营、客服、权益和风控处理，让后台人员不只是“看见用户分群”，还能对不同分群采取合适动作。

用户动作的核心定位：

| 模块   | 解决的问题               |
| ---- | ------------------- |
| 用户分群 | 圈出一批符合条件的目标用户       |
| 用户动作 | 定义可执行的运营、客服、权益、风控动作 |
| 执行动作 | 将一个未执行动作应用到指定分群     |
| 动作复盘 | 查看动作执行人数、排除人数、状态和效果 |

简单理解：

1. 用户分群回答“要对谁做”。
2. 用户动作回答“要做什么”。
3. 执行动作回答“把哪个动作应用到哪个分群”。
4. 效果复盘回答“做完之后效果如何”。

## 三、核心业务流程

### 3.1 标准流程

| 步骤  | 页面   | 操作           | 结果                |
| --- | ---- | ------------ | ----------------- |
| 1   | 用户分群 | 创建或选择一个分群    | 得到目标用户范围          |
| 2   | 用户动作 | 新建动作         | 生成一个状态为“未执行”的动作方案 |
| 3   | 用户分群 | 点击分群行内“执行动作” | 当前页打开执行动作弹窗       |
| 4   | 用户分群 | 选择一个“未执行”动作  | 动作绑定当前分群并执行       |
| 5   | 用户动作 | 查看动作列表       | 动作状态进入“执行中”或后续状态  |
| 6   | 用户动作 | 查看效果         | 复盘执行人数、排除人数、转化效果  |

### 3.2 关键产品规则

| 规则 | 说明 |
| --- | --- |
| 新建动作不选择分群 | 用户动作页只负责创建动作方案，不直接选择目标分群 |
| 执行动作必须从分群进入 | 在用户分群页选择某个分群后，才能执行动作 |
| 只能执行未执行动作 | 用户分群页的执行动作弹窗只能选择状态为“未执行”的动作 |
| 已执行动作不可再次作为新动作执行 | 避免同一动作重复执行导致数据复盘混乱 |
| 未执行动作可编辑、删除 | 动作尚未执行前允许调整内容或删除 |
| 执行中、已完成、待复盘动作不可编辑删除 | 已进入执行或复盘链路，应保持记录稳定 |

## 四、用户动作页面需求

### 4.1 页面目标

用户动作页用于管理动作方案和查看动作执行记录。运营、客服、风控人员可以在此创建动作、查看动作状态、编辑未执行动作、删除未执行动作，并查看已执行动作的效果。

### 4.2 页面布局

页面由四部分组成：

| 区域     | 说明                           |
| ------ | ---------------------------- |
| 顶部统计卡片 | 展示动作总数、执行中、待复盘、已完成           |
| 筛选区    | 按动作名称、动作类型、目标分群、动作状态、执行时间筛选  |
| 工具区    | 展示标题“用户动作”和说明，下方左侧展示“新建动作”按钮 |
| 列表区    | 展示动作记录和操作入口                  |

### 4.3 顶部统计卡片

| 卡片 | 统计口径 |
| --- | --- |
| 动作总数 | 当前列表内所有动作数量 |
| 执行中 | 状态为“执行中”的动作数量 |
| 待复盘 | 状态为“待复盘”的动作数量 |
| 已完成 | 状态为“已完成”的动作数量 |

### 4.4 筛选条件

| 筛选项 | 说明 |
| --- | --- |
| 动作名称 | 支持按动作名称模糊查询 |
| 动作类型 | 消息触达、权益发放、人工任务、风控处理 |
| 目标分群 | 筛选已绑定过目标分群的动作 |
| 动作状态 | 未执行、草稿、执行中、已完成、待复盘、已取消 |
| 执行时间 | 筛选指定时间范围内执行的动作 |

### 4.5 列表字段

| 字段   | 说明                       |
| ---- | ------------------------ |
| 动作名称 | 动作方案名称                   |
| 动作类型 | 消息触达、权益发放、人工任务、风控处理      |
| 目标分群 | 未执行动作显示“-”；已执行动作展示执行目标分群 |
| 执行人数 | 实际或预计执行用户数               |
| 排除人数 | 因账号异常、风险、注销等原因被排除的人数     |
| 状态   | 当前动作状态                   |
| 创建人  | 创建该动作的后台人员               |
| 执行时间 | 未执行动作显示“未执行”；已执行动作展示执行时间 |
| 效果   | 未执行显示“待选择分群执行”；执行后展示复盘指标 |
| 操作   | 查看、编辑、删除、取消              |

### 4.6 动作状态

| 状态 | 含义 | 可执行操作 |
| --- | --- | --- |
| 未执行 | 动作已创建，但尚未绑定分群执行 | 查看、编辑、删除 |
| 草稿 | 动作暂存，尚未完成配置 | 查看、编辑、删除 |
| 执行中 | 动作已绑定分群并开始执行 | 查看、取消 |
| 已完成 | 动作执行完成，效果已统计 | 查看 |
| 待复盘 | 动作已执行完成，但需要运营复盘 | 查看 |
| 已取消 | 动作被人工取消 | 查看 |

### 4.7 新建动作弹窗

新建动作用于创建一个可被分群执行的动作方案。新建动作时不选择目标分群，保存后动作状态为“未执行”。

字段要求：

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 动作类型 | 是 | 下拉选择消息触达、权益发放、人工任务、风控处理 |
| 创建方式 | 是 | 系统预设模板、自定义动作 |
| 系统预设 | 条件必填 | 选择系统预设模板时必填 |
| 动作名称 | 是 | 默认可由预设模板带出，也可手动修改 |
| 动作内容 | 条件必填 | 自定义动作时必填 |
| 复盘指标 | 否 | 自定义动作建议填写，如回流率、领取率、完成率 |
| 动作备注 | 否 | 说明动作目的、适用场景、注意事项 |

新建动作不应包含：

| 不展示字段 | 原因 |
| --- | --- |
| 目标分群 | 分群应在用户分群页执行时确定 |
| 目标用户 | 目标用户范围来自分群 |
| 排除规则 | 排除规则属于执行动作环节 |
| 动作有效期 | 当前版本暂不要求动作有效期 |

### 4.8 编辑动作

编辑动作用于修改未执行动作。

产品规则：

1. 只有“未执行”动作允许编辑。
2. 编辑时回填动作类型、创建方式、系统预设或自定义内容、动作名称、复盘指标和动作备注。
3. 已经执行过的动作不允许编辑，避免影响执行记录和复盘数据。

### 4.9 删除动作

删除动作用于移除未执行动作。

产品规则：

1. 只有“未执行”动作允许删除。
2. 删除前必须二次确认。
3. 已执行、执行中、待复盘、已完成动作不可删除。

## 五、动作类型设计

用户动作固定分为四类：

| 动作类型 | 使用场景 | 主要用户 |
| --- | --- | --- |
| 消息触达 | 召回、引导、活动通知、关系链激活 | 运营 |
| 权益发放 | 礼包、身份、优惠、活动资格、专属服务 | 运营 |
| 人工任务 | 客服回访、运营维护、人工核查 | 客服、运营 |
| 风控处理 | 风险复核、限制处理、观察名单处理 | 风控 |

动作类型选择规则：

1. 如果目的是让用户回来、参与活动、完成首聊，选择“消息触达”。
2. 如果目的是给用户奖励、身份、礼包、资格，选择“权益发放”。
3. 如果需要人工跟进、电话回访、客服处理，选择“人工任务”。
4. 如果用户存在风险、投诉、设备异常、违规行为，选择“风控处理”。

## 六、预设动作设计

### 6.1 消息触达类

| 预设动作 | 适用分群 | 动作内容 | 推荐复盘指标 |
| --- | --- | --- | --- |
| 沉默用户福利召回 | 沉默用户、流失预警用户、高价值沉默用户 | 好久不见，回来看看好友动态，还有一份限时权益待领取 | 回流率、登录率、消息互动率 |
| 新用户首聊引导 | 新用户、首聊未完成用户、资料未完善用户 | 完善资料并发送第一条消息，快速建立你的社交关系 | 首聊率、资料完善率、次日留存 |
| 活动邀请通知 | 社交活跃用户、活动目标用户、核心用户 | 你已获得本次活动优先参与资格，点击查看活动详情 | 活动参与率、点击率、转化率 |
| 关系链激活提醒 | 好友少、群聊少、互动低用户 | 添加好友或加入群聊，可以更快获得互动反馈 | 加好友率、群聊参与率 |

命中建议：

| 分群信号 | 推荐动作 |
| --- | --- |
| 近 7 日无登录 | 沉默用户福利召回 |
| 注册后未发消息 | 新用户首聊引导 |
| 活跃但近期参与下降 | 活动邀请通知 |
| 好友数低于平台均值 | 关系链激活提醒 |

### 6.2 权益发放类

| 预设动作 | 适用分群 | 动作内容 | 推荐复盘指标 |
| --- | --- | --- | --- |
| 高价值用户专属权益 | 高价值用户、核心用户、重点服务客户 | 发放 VIP 标识、专属客服或活动优先资格 | 领取率、活跃保持率、7 日留存 |
| 回流专属礼包 | 高价值沉默用户、流失预警用户 | 发放限时回流礼包，促进用户重新建立互动 | 回流率、回流后 7 日留存 |
| 新用户成长礼包 | 新用户、成长期用户 | 完成首聊、加好友、入群后获得成长权益 | 激活率、任务完成率 |
| 重点维护权益 | 重点服务客户、投诉后安抚用户 | 发放补偿权益或专属服务资格 | 投诉下降率、满意度 |

命中建议：

| 分群信号 | 推荐动作 |
| --- | --- |
| 高活跃且低风险 | 高价值用户专属权益 |
| 曾高活跃但近期沉默 | 回流专属礼包 |
| 注册 0-7 天且未完成关键行为 | 新用户成长礼包 |
| 客服标记重点跟进 | 重点维护权益 |

### 6.3 人工任务类

| 预设动作 | 适用分群 | 动作内容 | 推荐复盘指标 |
| --- | --- | --- | --- |
| 客服重点回访 | 投诉关注用户、客服重点回访池 | 生成客服回访任务，跟进投诉、问题闭环和满意度 | 回访完成率、满意度、投诉关闭率 |
| 运营重点维护 | VIP 用户、重点服务客户、核心用户 | 生成运营维护任务，记录维护结论和后续动作 | 任务完成率、活跃保持率 |
| 新用户人工引导 | 新用户、激活失败用户 | 由运营或客服进行新手引导和使用答疑 | 激活率、首聊率 |
| 异常用户人工核查 | 行为异常但风险未确认用户 | 人工核查用户资料、行为、投诉和设备情况 | 核查完成率、风险确认率 |

命中建议：

| 分群信号 | 推荐动作 |
| --- | --- |
| 被投诉或客服备注异常 | 客服重点回访 |
| 高价值用户活跃下降 | 运营重点维护 |
| 新用户未完成激活 | 新用户人工引导 |
| 多项异常但未达到高风险 | 异常用户人工核查 |

### 6.4 风控处理类

| 预设动作 | 适用分群 | 动作内容 | 推荐复盘指标 |
| --- | --- | --- | --- |
| 风险用户复核 | 中风险、高风险、设备复用观察池 | 生成风控复核任务，确认是否继续观察、限制或解除 | 风险确认率、误判率 |
| 高风险限制处理 | 高风险用户、封禁设备关联用户 | 执行限制加好友、限制群发、禁用设备等处理前确认 | 风险下降率、违规复发率 |
| 投诉集中核查 | 投诉关注用户、举报增长用户 | 核查投诉、举报、拉黑、敏感行为记录 | 投诉成立率、处理完成率 |
| 设备复用观察 | 设备复用用户、异常登录用户 | 对同设备多账号、频繁换设备用户加入观察 | 设备复用下降率 |

命中建议：

| 分群信号       | 推荐动作    |
| ---------- | ------- |
| 风险等级为高     | 高风险限制处理 |
| 风险等级为中     | 风险用户复核  |
| 近 30 日投诉增加 | 投诉集中核查  |
| 同设备关联多个账号  | 设备复用观察  |

## 七、分群预设设计

### 7.1 分群类型

| 分群类型 | 说明 | 适用场景 |
| --- | --- | --- |
| 静态分群 | 通过人工选择或导入用户形成固定名单 | 客服回访、重点维护、活动白名单 |
| 动态分群 | 根据规则自动命中用户，人数可随规则变化 | 召回、激活、风控观察、活跃分层 |

### 7.2 生命周期分群预设

| 预设分群 | 分群规则 | 推荐动作 |
| --- | --- | --- |
| 新用户激活池 | 注册 0-7 天，未完成首聊或资料不完整 | 新用户首聊引导、新用户成长礼包、新用户人工引导 |
| 成长期用户维护池 | 注册 8-30 天，近 7 日至少登录 2 天 | 活动邀请通知、新用户成长礼包 |
| 稳定活跃用户池 | 注册超过 30 天，近 30 日有稳定登录或互动 | 活动邀请通知、高价值用户专属权益 |
| 7 日沉默待召回 | 近 7 日无登录或近 14 日无有效互动 | 沉默用户福利召回、回流专属礼包 |
| 流失预警用户池 | 近 30 日无登录，但历史曾连续活跃超过 7 天 | 沉默用户福利召回、回流专属礼包 |

### 7.3 活跃价值分群预设

| 预设分群 | 分群规则 | 推荐动作 |
| --- | --- | --- |
| 核心用户维护池 | 活跃度大于等于 80，风险低，好友数、群聊数或消息量高于均值 | 高价值用户专属权益、运营重点维护 |
| 高价值沉默用户 | 曾为核心用户或高价值用户，近 7-30 日活跃下降 | 回流专属礼包、运营重点维护 |
| 社交活跃用户池 | 近 30 日消息互动、好友互动、群聊参与较高 | 活动邀请通知、高价值用户专属权益 |
| 关系链不足用户 | 好友数或群聊参与低于平台均值，但仍有登录 | 关系链激活提醒、新用户人工引导 |

### 7.4 客服运营分群预设

| 预设分群 | 分群规则 | 推荐动作 |
| --- | --- | --- |
| 重点服务客户 | 运营或客服人工标记为重点维护对象 | 高价值用户专属权益、运营重点维护 |
| 客服重点回访池 | 存在投诉、售后、反馈、人工备注待跟进 | 客服重点回访、重点维护权益 |
| 投诉后安抚池 | 投诉成立或客服判定需要安抚 | 重点维护权益、客服重点回访 |
| 活动目标用户池 | 满足活动参与条件或活动偏好标签 | 活动邀请通知、权益发放 |

### 7.5 风控分群预设

| 预设分群 | 分群规则 | 推荐动作 |
| --- | --- | --- |
| 高风险高活跃用户 | 风险等级高，同时活跃度高 | 高风险限制处理、风险用户复核 |
| 中风险观察池 | 风险等级中，命中多项轻中度异常 | 风险用户复核、异常用户人工核查 |
| 设备复用观察池 | 同一设备关联多个用户且超过设定阈值 | 设备复用观察、风险用户复核 |
| 异常登录用户池 | 短时间跨地区、跨设备或登录失败次数异常 | 风险用户复核、设备复用观察 |
| 投诉关注用户池 | 近 30 日被投诉、举报、拉黑次数较高 | 投诉集中核查、客服重点回访 |

## 八、分群执行动作需求

### 8.1 入口

入口位于用户分群列表操作列，按钮名称为“执行动作”。

点击后不跳转页面，应在当前用户分群页打开“执行动作”弹窗。

### 8.2 执行动作弹窗字段

| 字段 | 说明 |
| --- | --- |
| 目标分群 | 自动展示当前操作的分群名称 |
| 分群类型 | 静态分群或动态分群 |
| 当前成员 | 当前分群成员数 |
| 预计执行 | 扣除排除规则后的预计执行人数 |
| 选择动作 | 只能选择状态为“未执行”的动作 |
| 动作类型 | 选择动作后展示，不可单独修改 |
| 动作内容 | 选择动作后展示，不可单独修改 |
| 复盘指标 | 选择动作后展示 |
| 目标用户 | 当前分群全部成员、按当前筛选结果 |
| 排除规则 | 排除禁用、注销或账号异常用户；排除高风险用户 |
| 执行备注 | 记录本次执行目的、注意事项或处理原因 |

### 8.3 执行动作规则

| 规则 | 说明 |
| --- | --- |
| 只能选择未执行动作 | 下拉选项仅展示状态为“未执行”的动作 |
| 分群自动带入 | 当前点击的分群自动作为目标分群 |
| 分群不可手动切换 | 避免误操作到其他分群 |
| 风控处理可不排除高风险用户 | 风控类动作本身就是处理风险用户 |
| 非风控动作默认排除高风险用户 | 避免对高风险用户发放权益或活动触达 |
| 禁用、注销、账号异常用户默认排除 | 避免无效触达和异常执行 |

### 8.4 执行后状态变化

| 执行前 | 执行后 |
| --- | --- |
| 未执行 | 执行中 |

执行后应产生以下结果：

1. 动作绑定当前目标分群。
2. 动作记录执行人数和排除人数。
3. 动作记录执行时间。
4. 动作进入“执行中”状态。
5. 后续可进入“待复盘”或“已完成”。

## 十、效果复盘口径

### 10.1 通用指标

| 指标 | 说明 |
| --- | --- |
| 分群人数 | 执行动作时目标分群的当前成员数 |
| 排除人数 | 因排除规则未执行的人数 |
| 执行人数 | 分群人数减排除人数 |
| 执行成功数 | 实际完成触达、发放、任务生成或处理的人数 |
| 执行失败数 | 因账号异常、状态变化、系统限制等未成功的人数 |
| 执行完成率 | 执行成功数 / 执行人数 |

### 10.2 不同动作类型指标

| 动作类型 | 核心复盘指标 |
| --- | --- |
| 消息触达 | 送达率、点击率、登录率、回流率、消息互动率 |
| 权益发放 | 发放成功率、领取率、权益使用率、留存率 |
| 人工任务 | 任务生成数、任务完成率、回访完成率、满意度 |
| 风控处理 | 复核完成率、风险确认率、限制成功率、风险下降率 |

### 10.3 状态流转建议

| 状态 | 进入条件 |
| --- | --- |
| 未执行 | 动作新建完成，尚未被分群执行 |
| 执行中 | 分群页确认执行动作后进入 |
| 待复盘 | 动作执行完成，但效果数据尚未确认 |
| 已完成 | 效果复盘完成 |
| 已取消 | 执行中动作被人工取消 |

## 十二、产品优化建议

1. 后续可将“分群预设”和“动作预设”做成可维护配置，支持运营人员新增、停用、排序。
2. 执行动作时可根据分群名称、规则摘要和风险状态自动推荐动作，减少运营选择成本。
3. 用户动作列表后续建议增加“复制动作”，方便复用成熟动作方案。
4. 分群执行动作后建议形成执行记录，用于分群详情页展示历史操作。
5. 待复盘动作应提供复盘入口，要求运营填写结论，如继续执行、调整动作、停止动作。
6. 对风控动作建议增加更明显的确认提示，避免误限制正常用户。
7. 对权益发放动作建议显示预计成本，避免大规模分群误发权益。
