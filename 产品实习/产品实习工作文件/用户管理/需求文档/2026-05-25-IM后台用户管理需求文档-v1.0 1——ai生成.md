# IM后台用户管理产品需求文档

## 一、文档说明

本文档用于说明 IM 后台「用户管理」相关产品需求，面向产品、前端、测试、运营、客服、风控等角色统一理解页面能力、业务规则、指标口径和验收标准。

本文档不包含后端接口、数据库设计、代码实现、技术权限点等内容。所有标签、评分、风险等级和统计口径先按 IM 社交类产品通用运营规则设计，后续可根据真实业务数据进行阈值调优。

## 二、产品目标

用户管理模块用于统一管理平台用户资产，帮助后台人员完成用户查询、用户识别、用户分层、标签打标、分组运营、风险发现和风险处置。

核心目标如下：

1. 提供完整的用户检索、查看、编辑、启停、导出、登录记录、好友关系和风控操作能力。
2. 建立统一的用户标签体系，支持生命周期、活跃度、用户价值、风险状态等维度识别用户。
3. 建立用户活跃度评分和风险等级判断标准，帮助运营、客服、风控快速判断用户状态。
4. 通过用户画像和可视化图表展示用户规模、结构、趋势、关系和风险变化。
5. 通过用户分组承接运营、客服、风控动作，让标签识别结果可以落到具体人群管理。

## 三、用户管理功能范围

### 3.1 用户列表

用户列表是用户管理的主入口，用于快速查找用户、查看用户状态并执行基础管理动作。

页面顶部展示核心统计卡片：

| 指标 | 说明 |
| --- | --- |
| 用户总量 | 当前有效注册用户总数 |
| 今日新增用户数 | 当天新增注册用户数 |
| 今日登录用户数 | 当天至少成功登录一次的去重用户数 |
| 当前在线人数 | 当前仍处于在线会话状态的用户数 |
| 黑名单用户总数 | 当前处于黑名单、封禁或强风控状态的用户数 |

列表支持筛选：

| 筛选项 | 说明 |
| --- | --- |
| 关键词 | 支持按昵称、手机号、用户 ID 查询 |
| 在线状态 | 全部、在线、离线 |
| 用户类型 | 全部、外部用户、内部用户 |
| 用户状态 | 全部、启用、禁用、登录锁定 |
| 风险等级 | 全部、低风险、中风险、高风险 |
| 用户标签 | 支持按系统标签或人工标签筛选 |
| 用户分组 | 支持按用户所在分组筛选 |

列表展示字段：

| 字段 | 说明 |
| --- | --- |
| 用户信息 | 昵称、手机号、性别标识 |
| 用户 ID | 用户唯一业务标识 |
| 用户类型 | 外部用户、内部用户 |
| 登录锁定状态 | 已锁定、未锁定 |
| 在线状态 | 在线、离线 |
| 是否启用 | 启用、禁用 |
| 登录日志 | 用户登录记录次数，点击查看明细 |
| 登录 IP | 最近一次登录 IP |
| 注册时间 | 用户注册时间 |
| 操作 | 编辑、重置密码、好友管理、禁止加好友、拉黑 IP、禁用设备等 |

用户列表应支持以下操作：

| 操作 | 产品规则 |
| --- | --- |
| 新增用户 | 创建用户基础信息，至少包含手机号、昵称、性别、用户类型等 |
| 编辑用户 | 修改用户基础信息，用户类型原则上不建议频繁变更 |
| 删除用户 | 删除后用户不可继续登录，操作前需明确提示影响 |
| 批量删除 | 仅对勾选用户生效，需展示影响人数 |
| 导出 | 导出当前筛选条件下的用户列表 |
| 启用/禁用 | 禁用后用户不可登录或使用核心功能 |
| 解锁登录 | 用户因登录失败或安全策略被锁定后，可手动解除 |
| 重置密码 | 管理员为用户重置登录密码 |
| 登录日志 | 查看登录 IP、登录时间、登录设备、登录状态 |
| 好友管理 | 查看用户好友列表、好友 ID、建立时间 |
| 禁止/允许加好友 | 控制用户是否可发起新增好友行为 |
| 拉黑/解封 IP | 将用户最近登录 IP 纳入或移出黑名单 |
| 禁用/解封设备 | 将用户最近登录设备纳入或移出封禁范围 |

### 3.2 用户详情

用户详情页用于形成单个用户的完整档案，帮助运营、客服、风控在同一页面理解用户状态。

用户详情应展示以下内容：

| 区域     | 展示内容                               |
| ------ | ---------------------------------- |
| 用户身份   | 头像、昵称、用户 ID、在线状态、用户类型              |
| 基础资料   | 手机号、注册时间、来源渠道、地区、最近登录 IP、常用设备      |
| 状态标签   | 账号状态、实名状态、风险等级、活跃等级、生命周期标签         |
| 社交信息   | 好友数、加入群数、联系人互动、群聊参与                |
| 行为趋势   | 近 30 日消息量、登录频次、在线活跃趋势              |
| 关键指标   | 近 7 日发消息、近 7 日登录、近 7 日加好友、近 7 日举报数 |
| 标签摘要   | 当前命中的系统标签和人工标签                     |
| 风险摘要   | 风险等级、异常次数、敏感词命中、设备异常、投诉记录          |
| 最近事件流  | 登录、打标、分组、风控、客服处理等关键事件              |
| 最近处理记录 | 管理员、客服、运营、风控对该用户执行过的处理动作           |

用户详情应满足以下产品要求：

1. 从用户列表、用户画像、标签命中用户、分组成员等位置进入详情时，应展示对应用户的真实档案。
2. 详情页应突出「用户当前是否正常」「是否值得运营」「是否存在风险」「下一步建议动作」。
3. 用户标签、分组、风险记录、处理记录应形成时间线，便于追踪变化原因。

用户详情页建议按「基础档案 + 行为分析 + 社交关系 + 风险判断 + 处理记录」组织，避免只展示静态资料。

详情页核心分析维度：

| 分析维度 | 需要回答的问题 | 关键判断 |
| --- | --- | --- |
| 身份真实性 | 该用户资料是否完整、身份是否可信 | 手机号、头像、昵称、实名或必要资料是否完整 |
| 当前状态 | 用户当前是否可正常使用 | 在线状态、启用状态、登录锁定、风险限制 |
| 活跃价值 | 用户是否值得重点运营 | 活跃度评分、消息互动、好友关系、群聊参与 |
| 生命周期 | 用户处于哪个成长阶段 | 新用户、成长期、成熟期、稳定期、沉默期、流失预警 |
| 社交质量 | 用户关系链是否健康 | 好友数、有效互动人数、群聊参与、联系人互动率 |
| 风险水平 | 用户是否存在安全或违规风险 | 风险等级、设备异常、IP 异常、投诉举报、敏感行为 |
| 处理闭环 | 用户是否有未完成事项 | 客服备注、运营跟进、风控处理、最近事件 |

详情页建议展示的操作按钮：

| 按钮 | 使用场景 | 产品说明 |
| --- | --- | --- |
| 编辑资料 | 基础资料需要修正 | 适合后台人员修正昵称、手机号、头像等基础信息 |
| 重置密码 | 用户无法登录或安全处理需要 | 属于账号安全动作，应有明确确认 |
| 打标签 | 需要人工标记用户特征 | 适合标记 VIP、重点跟进、投诉用户、召回候选等 |
| 加入分组 | 需要进入运营、客服或风控池 | 适合加入召回池、回访池、观察名单 |
| 好友管理 | 需要查看关系链 | 用于客服排查、风控核查、关系质量分析 |
| 风控处理 | 用户存在中高风险 | 包含限制加好友、拉黑 IP、禁用设备、加入观察名单等 |
| 添加备注 | 需要记录人工判断 | 适合客服、运营、风控记录跟进结论 |

不同用户状态下的主按钮建议：

| 用户状态      | 主按钮建议            |
| --------- | ---------------- |
| 正常高价值用户   | 打标签、加入分组、添加备注    |
| 沉默或流失预警用户 | 加入召回池、打标签、添加备注   |
| 中风险用户     | 风控处理、加入观察名单、添加备注 |
| 高风险用户     | 风控处理、限制行为、添加备注   |
| 投诉关注用户    | 加入回访池、添加备注、风控处理  |

用户详情页图表应围绕单个用户的变化趋势和结构分析，而不是展示平台整体数据。

详情页建议图表：

| 图表 | 图表类型 | 展示内容 | 产品价值 |
| --- | --- | --- | --- |
| 近 30 天行为趋势 | 折线图 | 消息量、登录次数、在线活跃、互动人数变化 | 判断用户活跃是在上升、稳定还是下降 |
| 活跃度评分环形图 | 环形图 | 当前综合活跃评分及分层 | 快速识别用户活跃水平 |
| 社交关系概览 | 进度条或条形图 | 一对一会话、群聊会话、系统通知互动占比 | 判断用户主要互动方式 |
| 近 7 日关键行为 | 指标卡 | 发消息数、登录天数、加好友数、举报数 | 快速查看近期行为强弱 |
| 风险构成 | 雷达图或分组条形图 | 设备风险、登录风险、行为风险、投诉风险、资料风险 | 解释风险等级来源 |
| 事件时间线 | 时间轴 | 登录、打标、分组、投诉、限制、解除等事件 | 还原用户状态变化过程 |

用户详情页图表解读规则：

1. 行为趋势连续下降时，应提示「活跃下降」或「流失预警」。
2. 活跃度高但风险等级也高时，应优先展示风险提醒，避免只按活跃判断用户价值。
3. 社交关系低但登录频繁时，应提示「关系链不足」，建议引导添加好友或参与群聊。
4. 投诉、举报、设备异常等事件应在时间线上明显标识，便于客服和风控追溯。
5. 详情页所有图表都应支持查看指标说明，避免运营人员误读数据。

### 3.3 用户画像

用户画像用于从整体视角分析用户结构、活跃情况、生命周期、关系质量和风险信号。

筛选项：

| 筛选项 | 说明 |
| --- | --- |
| 统计周期 | 支持自定义起止日期 |
| 周期快捷项 | 今日、近 7 日、近 30 日 |
| 来源 | 全部来源、Android、iOS、Web、开放平台等 |
| 地区 | 全部地区或指定地区 |

核心指标卡：

| 指标 | 说明 |
| --- | --- |
| 活跃用户 | 统计周期内活跃度评分大于等于 50 的用户数 |
| 核心用户 | 满足核心用户标签规则的用户数 |
| 风险用户 | 风险等级为中风险或高风险的用户数 |
| 资料完整度 | 用户资料完整字段数占应填写字段数的比例 |

画像图表：

| 图表 | 统计内容 |
| --- | --- |
| 用户分层 | 高价值用户、稳定活跃用户、沉默召回用户、社交活跃用户、安全观察用户 |
| 生命周期分布 | 新用户、成长期、成熟期、稳定期、沉默期人数及占比 |
| 近 7 日活跃趋势 | 按日展示新增用户、核心用户、沉默用户、风险用户变化 |
| 关系洞察 | 好友关系覆盖率、群聊参与率、联系人互动率 |
| 风险信号 | 设备复用、异常登录、资料缺失、投诉增多等风险类型 |
| 用户明细 | 展示命中画像筛选条件的用户列表 |

用户画像应提供可行动建议：

| 信号 | 建议动作 |
| --- | --- |
| 沉默用户占比上升 | 建议进入召回分组，进行消息触达或活动召回 |
| 高风险用户上升 | 建议风控复核，并观察设备、IP、投诉来源 |
| 资料完整度下降 | 建议引导用户补全头像、手机号、实名或必要资料 |
| 核心用户增长放缓 | 建议分析活跃路径和社交关系质量 |
| 设备复用增加 | 建议加强新设备登录校验和设备关联排查 |

### 3.4 用户标签

用户标签用于描述用户特征，回答「这个用户是谁、有什么状态、有什么价值或风险」。

标签分为两类：

| 类型 | 说明 |
| --- | --- |
| 系统自动标签 | 根据规则自动命中，适合生命周期、活跃度、风险等动态特征 |
| 人工标签 | 由运营、客服、风控人员手动维护，适合重点跟进、投诉用户、VIP 用户等业务判断 |

标签管理支持：

| 功能 | 产品规则 |
| --- | --- |
| 标签统计 | 展示标签总数、启用标签数、停用标签数、系统标签数、人工标签数 |
| 标签查询 | 按标签名称、标签分类、打标方式、标签状态筛选 |
| 新建标签 | 设置标签名称、分类、打标方式、状态、说明和命中规则 |
| 编辑标签 | 可修改说明、状态、人工命中用户或自动规则 |
| 启用/停用 | 停用后不再新增命中，历史命中记录保留 |
| 删除标签 | 已命中用户的标签不建议直接删除，应优先停用 |
| 命中用户 | 查看命中该标签的用户清单 |
| 人工打标 | 支持手动选择用户或导入用户 ID、手机号 |
| 自动打标 | 支持按注册时间、活跃度、风险状态、账号状态、渠道来源等规则打标 |

### 3.5 用户分组

用户分组用于承接运营、客服、风控动作，回答「要对这批用户做什么」。

分组类型：

| 类型 | 说明 |
| --- | --- |
| 静态分组 | 成员由人工选择或导入，适合客服重点回访、VIP 维护、临时活动名单 |
| 动态分组 | 成员由规则自动命中，适合沉默召回、高风险观察、开放渠道新增用户等 |

分组管理支持：

| 功能 | 产品规则 |
| --- | --- |
| 分组统计 | 展示分组总数、静态分组数、动态分组数、启用分组数、今日刷新数 |
| 分组查询 | 按分组名称、分组类型、分组状态、分组来源筛选 |
| 新增分组 | 设置分组名称、类型、说明、成员或规则 |
| 编辑分组 | 修改分组名称、说明、静态成员或动态规则 |
| 启用/停用 | 停用后该分组不再参与运营动作或自动刷新 |
| 批量停用 | 对选中分组批量停用 |
| 成员详情 | 查看分组内成员、加入时间、加入来源 |
| 静态成员维护 | 支持手动添加、导入、移除成员 |
| 动态分组刷新 | 按规则重新计算命中成员 |
| 导出 | 导出当前筛选条件下的分组或成员 |

预设分组建议：

| 分组名称 | 类型 | 规则或用途 |
| --- | --- | --- |
| 重点服务客户 | 静态分组 | 客服或运营手动维护的高价值客户名单 |
| 高风险高活跃用户 | 动态分组 | 风险等级为高，且活跃度为高活跃 |
| 7 日沉默待召回 | 动态分组 | 近 7 日无登录或近 14 日无有效互动 |
| 客服重点回访池 | 静态分组 | 投诉、退款、问题未闭环用户 |
| 开放渠道新增用户 | 动态分组 | 来源为开放平台，注册时间在近 30 日 |

### 3.6 风控管理

风控管理用于集中管理黑名单、风险 IP、封禁设备和异常用户。

风控对象：

| 对象 | 说明 |
| --- | --- |
| 黑名单用户 | 被禁用、封禁或强风控限制的用户 |
| IP 黑名单 | 被认定为不可登录或高危访问来源的 IP |
| IP 风险库 | 存在异常行为但未必直接封禁的风险 IP |
| 设备封禁 | 被禁止继续登录或使用的设备 |
| 异常登录 | 异地、异设备、高频失败、异常时间段登录等 |
| 投诉举报 | 被用户投诉、举报或客服标记异常的对象 |

风控管理应关注处理闭环：

| 环节 | 说明 |
| --- | --- |
| 风险来源 | 明确风险来自设备、IP、登录、行为、投诉还是人工标记 |
| 风险等级 | 低风险、中风险、高风险 |
| 处理动作 | 观察、提醒、限制、封禁、解封 |
| 处理原因 | 必须记录操作原因，便于复盘 |
| 处理状态 | 待处理、处理中、已处理、已解除 |
| 处理记录 | 记录处理人、处理时间、处理结论 |

## 四、预设标签体系与命中规则

### 4.1 标签分类

建议预设以下标签分类：

| 分类     | 用途                         |
| ------ | -------------------------- |
| 生命周期标签 | 描述用户处于注册、成长、成熟、稳定、沉默、流失等阶段 |
| 活跃度标签  | 描述用户当前活跃水平                 |
| 价值标签   | 识别核心用户、高价值用户和重点跟进用户        |
| 社交标签   | 描述好友、群聊、互动关系特征             |
| 资料标签   | 描述用户资料完整情况                 |
| 来源标签   | 描述用户注册或登录来源                |
| 风控标签   | 描述用户风险状态和风险原因              |
| 客服标签   | 描述客服处理和跟进状态                |
| 运营标签   | 描述活动、召回、转化、重点维护等运营状态       |

## 五、活跃度评分设计

### 5.1 评分原则

活跃度评分用于衡量用户近期使用深度和关系活跃程度。评分总分 100 分，默认按近 30 日行为计算。分值越高，表示用户越活跃、越稳定。

评分应遵循以下原则：

1. 既关注登录，也关注消息、好友、群聊等有效互动。
2. 最近行为权重高于较早行为。
3. 只登录但无互动不应被判定为高活跃。
4. 存在明显风险行为时，应对活跃评分进行适度扣分，避免高风险用户被误判为优质用户。

### 5.2 评分构成

| 维度 | 分值 | 评分说明 |
| --- | ---: | --- |
| 登录活跃 | 25 | 根据近 30 日登录天数、连续登录天数、最近登录时间评分 |
| 消息互动 | 25 | 根据发消息次数、会话参与次数、有效互动人数评分 |
| 社交关系 | 20 | 根据好友数、新增好友数、群聊参与、联系人互动评分 |
| 留存稳定 | 15 | 根据注册时长、连续活跃周期、沉默间隔评分 |
| 资料完整 | 10 | 根据头像、昵称、手机号、实名或必要资料完整度评分 |
| 风险扣分 | -5 | 根据异常登录、投诉、设备异常、封禁记录进行扣分 |

### 5.3 登录活跃评分

| 条件 | 分值 |
| --- | ---: |
| 近 30 日登录天数大于等于 20 天 | 15 |
| 近 30 日登录天数 10-19 天 | 10 |
| 近 30 日登录天数 3-9 天 | 5 |
| 近 30 日登录天数小于 3 天 | 0 |
| 连续登录大于等于 7 天 | 5 |
| 连续登录 3-6 天 | 3 |
| 最近 24 小时内登录 | 5 |
| 最近 3 天内登录 | 3 |

登录活跃最高 25 分，超过部分不累计。

### 5.4 消息互动评分

| 条件 | 分值 |
| --- | ---: |
| 近 30 日发消息次数高于平台均值 150% | 10 |
| 近 30 日发消息次数达到平台均值 | 7 |
| 近 30 日有发消息但低于平台均值 | 3 |
| 近 30 日参与会话天数大于等于 15 天 | 6 |
| 近 30 日参与会话天数 5-14 天 | 4 |
| 近 30 日有效互动人数大于等于 10 人 | 6 |
| 近 30 日有效互动人数 3-9 人 | 4 |
| 近 30 日存在群聊或多人互动 | 3 |

消息互动最高 25 分，超过部分不累计。

### 5.5 社交关系评分

| 条件 | 分值 |
| --- | ---: |
| 好友数高于平台均值 | 5 |
| 近 30 日新增好友大于等于 5 人 | 5 |
| 加入群聊数大于等于 3 个 | 4 |
| 近 30 日群聊参与天数大于等于 5 天 | 3 |
| 联系人互动覆盖率高于平台均值 | 3 |

社交关系最高 20 分。

### 5.6 留存稳定评分

| 条件 | 分值 |
| --- | ---: |
| 注册超过 180 天且近 30 日仍活跃 | 5 |

留存稳定最高 15 分。

### 5.7 资料完整评分

## 六、风险等级判断

### 6.1 风险评分原则

风险评分用于衡量用户当前存在异常、违规、盗号、骚扰、营销、投诉等风险的可能性。风险评分总分 100 分，分值越高风险越高。

风险等级不仅看分数，也看是否命中强规则。强规则包括黑名单 IP、封禁设备、严重投诉成立、明显违规行为等。

### 6.2 风险评分构成

| 维度 | 分值 | 说明 |
| --- | ---: | --- |
| 设备风险 | 25 | 设备复用、频繁换设备、封禁设备再次登录 |
| 登录风险 | 20 | 异地登录、异常 IP、登录失败次数过多、高频登录 |
| 行为风险 | 25 | 高频加好友、高频群发、敏感词命中、异常消息行为 |
| 投诉风险 | 20 | 被举报、被拉黑、投诉成立、客服标记异常 |
| 资料风险 | 10 | 资料缺失、手机号异常、身份信息不一致 |

### 6.3 设备风险判断

| 条件 | 风险分 |
| --- | ---: |
| 同一设备关联用户数超过平台阈值 | 10 |
| 近 7 日频繁切换设备 | 5 |
| 使用曾被封禁设备登录 | 15 |
| 设备信息缺失或异常 | 5 |

设备风险最高 25 分。命中封禁设备可直接判定为高风险。

### 6.4 登录风险判断

| 条件 | 风险分 |
| --- | ---: |
| 短时间内跨地区登录 | 8 |
| 登录 IP 命中风险库 | 8 |
| 登录 IP 命中黑名单 | 20 |
| 短时间登录失败次数过多 | 6 |
| 非常用时间段高频登录 | 4 |

登录风险最高 20 分。命中黑名单 IP 可直接判定为高风险。

### 6.5 行为风险判断

| 条件 | 风险分 |
| --- | ---: |
| 短时间高频加好友 | 8 |
| 短时间发送大量相似消息 | 8 |
| 敏感词命中次数超过阈值 | 8 |
| 群聊刷屏或异常发言 | 6 |
| 被多个用户拉黑 | 5 |

行为风险最高 25 分。

### 6.6 投诉风险判断

| 条件 | 风险分 |
| --- | ---: |
| 近 30 日被投诉 | 5 |
| 投诉成立 | 10 |
| 近 30 日被举报次数超过阈值 | 8 |
| 客服人工标记异常 | 5 |
| 已有处罚记录 | 8 |

投诉风险最高 20 分。

### 6.7 资料风险判断

| 条件         | 风险分 |
| ---------- | --: |
| 未绑定手机号     |   3 |
| 头像、昵称等资料缺失 |   2 |
| 身份信息不一致    |   5 |
| 注册来源异常     |   3 |

资料风险最高 10 分。

### 6.8 风险等级分层

| 分数或规则                               | 风险等级 | 处理建议                        |
| ----------------------------------- | ---- | --------------------------- |
| 风险评分大于等于 70，或命中封禁设备、黑名单 IP、严重投诉等强规则 | 高风险  | 优先复核，可限制登录、限制加好友、封禁设备或加入黑名单 |
| 风险评分 40-69，存在多项轻中度异常                | 中风险  | 加入观察名单，必要时提醒或限制部分行为         |
| 风险评分小于 40，无明显异常或仅轻微资料缺失             | 低风险  | 正常使用，引导资料补全                 |

| 在线活跃 | 当前用户每日在线或活跃时长        | 用于观察真实使用深度    |
| 互动人数 | 当前用户每日发生有效互动的去重联系人数量 | 用于观察关系链是否扩大   |

趋势判断：

| 趋势 | 判断规则 | 产品解释 |
| --- | --- | --- |
| 活跃上升 | 近 7 日均值高于前 7 日均值 20% 及以上 | 用户活跃正在增强 |
| 活跃稳定 | 近 7 日均值较前 7 日变化在正负 20% 以内 | 用户使用状态稳定 |
| 活跃下降 | 近 7 日均值低于前 7 日均值 20% 及以上 | 用户可能进入沉默或流失 |
| 突发异常 | 单日行为量高于近 30 日日均值 3 倍以上 | 需要结合风险规则判断是否异常 |

#### 7.10.2 活跃度评分环形图

| 指标       | 统计口径                                   |
| -------- | -------------------------------------- |
| 综合活跃评分   | 按本文档「活跃度评分设计」计算的当前用户近 30 日评分           |
| 高于同类用户比例 | 当前用户评分超过同生命周期、同来源或同类型用户的比例             |
| 活跃等级     | 80-100 高活跃，50-79 中活跃，20-49 低活跃，0-19 沉默 |

环形图中间展示综合分，图表下方展示评分结论，例如「高于 78% 同期新用户」。

#### 7.10.3 近 7 日关键行为卡片

| 指标       | 统计口径               | 趋势说明          |
| -------- | ------------------ | ------------- |
| 近 7 日发消息 | 当前用户近 7 日发送消息总数    | 与上一个 7 日周期对比  |
| 近 7 日登录  | 当前用户近 7 日成功登录天数或次数 | 与上一个 7 日周期对比  |
| 近 7 日加好友 | 当前用户近 7 日新增好友数     | 可识别社交扩张或异常加好友 |
| 近 7 日举报数 | 当前用户近 7 日被举报或投诉次数  | 用于识别风险和客服关注   |

趋势文案建议：

| 变化 | 文案示例 |
| --- | --- |
| 明显上升 | 活跃提升、互动增强、关系增长 |
| 基本稳定 | 持续稳定、状态平稳 |
| 明显下降 | 活跃下降、关系减弱、存在流失风险 |
| 风险增加 | 投诉增加、举报增加、需关注 |

#### 7.10.4 社交关系概览

| 指标       | 统计口径                            | 说明           |
| -------- | ------------------------------- | ------------ |
| 一对一会话占比  | 当前用户一对一会话消息数 / 当前用户全部消息数        | 判断用户是否偏私聊互动  |
| 群聊会话占比   | 当前用户群聊消息数 / 当前用户全部消息数           | 判断用户是否偏群聊活跃  |
| 系统通知互动占比 | 当前用户点击、回复或处理系统通知次数 / 当前用户全部互动次数 | 判断用户是否主要被动响应 |
| 好友数      | 当前有效好友数量                        | 反映关系链规模      |
| 加入群数     | 当前加入的有效群聊数量                     | 反映群聊覆盖       |
| 联系人互动人数  | 近 30 日发生有效互动的去重联系人数量            | 反映真实社交质量     |

解读规则：

1. 一对一会话占比高，说明用户更依赖熟人关系。
2. 群聊会话占比高，说明用户更依赖群体互动。
3. 系统通知互动占比高但真人互动低，说明用户可能只是被动使用。
4. 好友数高但互动人数低，说明关系链可能质量不足。

#### 7.10.5 风险构成图

| 风险维度 | 统计口径                           |
| ---- | ------------------------------ |
| 设备风险 | 当前用户命中设备复用、频繁换设备、封禁设备等规则的风险分   |
| 登录风险 | 当前用户命中异常 IP、异地登录、登录失败频繁等规则的风险分 |
| 行为风险 | 当前用户命中高频加好友、群发异常、敏感词等规则的风险分    |
| 投诉风险 | 当前用户近 30 日投诉、举报、拉黑、客服异常标记的风险分  |
| 资料风险 | 当前用户资料缺失、身份信息异常、手机号异常的风险分      |

风险构成图用于解释「为什么该用户是低风险、中风险或高风险」。当某一维度风险明显高于其他维度时，页面应突出展示该风险原因和建议动作。

#### 7.10.6 事件时间线

| 事件类型   | 统计范围                | 展示内容           |
| ------ | ------------------- | -------------- |
| 登录事件   | 最近登录、异常登录、设备变化      | 时间、设备、IP、地区、结果 |
| 运营事件   | 打标签、加入分组、移出分组       | 时间、动作、操作人、说明   |
| 客服事件   | 添加备注、回访、投诉处理        | 时间、处理人、处理结论    |
| 风控事件   | 限制加好友、拉黑 IP、禁用设备、解封 | 时间、风险原因、处理结果   |
| 用户行为事件 | 发消息、加好友、群聊活跃、资料补全   | 时间、行为摘要、影响指标   |

时间线默认按时间倒序展示，重要风险事件应高亮。事件应帮助解释用户状态变化，而不是仅作为流水记录。

## 八、页面验收标准

### 8.1 用户列表验收

1. 能按关键词、在线状态、用户类型、用户状态、风险等级、标签、分组筛选用户。
2. 统计卡片展示用户总量、今日新增、今日登录、当前在线、黑名单用户总数。
3. 用户状态、在线状态、登录锁定状态含义清晰，不相互混淆。
4. 点击登录日志可查看登录 IP、时间、设备、登录结果。
5. 点击用户昵称可进入对应用户详情。
6. 批量删除、禁用、拉黑、设备封禁等高影响操作必须有明确确认提示。

### 8.2 用户详情验收

1. 能完整展示用户基础资料、账号状态、标签、分组、活跃评分、风险等级。
2. 能展示登录设备、最近 IP、好友数、群聊参与、最近行为。
3. 能展示该用户的标签摘要、风险摘要和最近事件流。
4. 风险等级和活跃等级需与评分规则保持一致。
5. 页面能帮助运营、客服、风控判断下一步处理动作。

### 8.3 用户画像验收

1. 能按周期、来源、地区筛选画像数据。
2. 活跃用户、核心用户、风险用户、资料完整度指标口径清晰。
3. 用户分层、生命周期、趋势、关系洞察、风险信号图表均有明确统计含义。
4. 风险信号必须展示建议动作，而不只是展示数字。
5. 用户明细可追溯到具体用户详情。

### 8.4 用户标签验收

1. 标签分类、打标方式、状态、命中用户数展示清晰。
2. 人工标签支持手动选择或导入用户。
3. 自动标签支持按生命周期、活跃度、风险、账号状态、来源等规则命中。
4. 标签停用后不再新增命中，历史记录保留。
5. 标签规则必须可解释，命中用户可查看。

### 8.5 用户分组验收

1. 静态分组和动态分组定位清晰。
2. 静态分组支持人工维护成员。
3. 动态分组支持按规则刷新成员。
4. 分组详情能展示成员、加入时间、加入来源、规则摘要。
5. 分组应能承接运营、客服、风控动作，而不仅是名单展示。

### 8.6 风控管理验收

1. 能区分黑名单用户、风险 IP、设备封禁、异常登录、投诉举报等风险对象。
2. 风险对象需展示风险来源、风险等级、处理状态和处理建议。
3. 高风险对象应优先展示并支持快速处理。
4. 解封或解除风险限制时，应记录原因，避免无依据解除。

## 九、产品提升建议

1. 将用户管理从「查用户」升级为「管用户」，增加批量打标签、加入分组、加入观察名单、进入召回池等动作入口。
2. 统一用户状态体系，避免在线状态、启用状态、登录锁定、黑名单、风险等级混淆。
3. 用户画像图表应提供行动建议，例如召回、复核、补全资料、限制行为等。
4. 明确标签和分组的定位：标签描述用户特征，分组承接业务动作。
5. 建立用户生命周期管理能力，支持从新用户、成长、成熟、稳定、沉默到召回的完整运营链路。
6. 用户详情应成为客服、运营、风控协作的统一工作台。
7. 风险处理应形成闭环，包含风险来源、处理动作、处理原因、处理状态和处理记录。
8. 标签和分组应增加效果评估，例如召回成功率、活跃提升、投诉下降、转化变化等。
9. 高影响操作应加强防误操作设计，展示影响人数、操作后果和解除方式。
10. 增加用户备注与协作记录，帮助多角色共同维护重点用户。
