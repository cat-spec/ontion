# 二、用户详细界面

用户详情应满足以下产品要求：

1. 从用户列表、用户画像、标签命中用户、分组成员等位置进入详情时，应展示对应用户的真实档案。
    
2. 详情页应突出「用户当前是否正常」「是否值得运营」「是否存在风险」「下一步建议动作」。
    
3. 用户标签、分组、风险记录、处理记录应形成时间线，便于追踪变化原因。
    


| 区域     | 展示内容                                |
| ------ | ----------------------------------- |
| 用户身份   | 头像、昵称、用户 ID、在线状态、用户类型               |
| 基础资料   | 手机号、注册时间、来源渠道、地区、最近登录 IP、常用设备       |
| 状态标签   | 账号状态、实名状态、风险等级、活跃等级、生命周期标签          |
| 好友关系网  | 可视化展示用户好友网络                         |
| 社交信息   | 好友数、加入群数、联系人互动、群聊参与                 |
| 行为趋势   | 近 30 日消息量、登录频次、在线活跃趋势               |
| 关键指标   | 近 7 日发消息、近 7 日登录、近 7 日加好友、近 7 日被举报数 |
| 标签摘要   | 当前命中的系统标签和人工标签                      |
| 风险摘要   | 风险等级、异常次数、敏感词命中、设备异常、投诉记录           |
| 最近事件流  | 登录、打标、分组、风控、客服处理等关键事件               |
| 最近处理记录 | 管理员、客服、运营、风控对该用户执行过的处理动作            |

# 四、用户标签

用户标签用于描述用户特征，回答「这个用户是谁、有什么状态、有什么价值或风险」。

标签分为两类：


| 类型     | 说明                                      |
| ------ | --------------------------------------- |
| 系统自动标签 | 根据规则自动命中，适合生命周期、活跃度、风险等动态特征             |
| 人工标签   | 由运营、客服、风控人员手动维护，适合重点跟进、投诉用户、VIP 用户等业务判断 |

标签管理支持： 


| 功能    | 产品规则                           |
| ----- | ------------------------------ |
| 标签统计  | 展示标签总数、启用标签数、停用标签数、系统标签数、人工标签数 |
| 标签查询  | 按标签名称、标签分类、打标方式、标签状态筛选         |
| 新建标签  | 设置标签名称、分类、打标方式、状态、说明和命中规则      |
| 编辑标签  | 可修改说明、状态、人工命中用户或自动规则           |
| 启用/停用 | 停用后不再新增命中，历史命中记录保留             |
| 删除标签  | 已命中用户的标签不建议直接删除，应优先停用          |
| 命中用户  | 查看命中该标签的用户清单                   |
| 人工打标  | 支持手动选择用户或导入用户 ID、手机号           |
| 自动打标  | 支持按注册时间、活跃度、风险状态、账号状态等规则打标     |

# 五、用户分群

用户分组用于承接运营、客服、风控动作，回答「要对这批用户做什么」。

分组类型：


| 类型   | 说明                                |
| ---- | --------------------------------- |
| 静态分组 | 成员由人工选择或导入，适合客服重点回访、VIP 维护、临时活动名单 |
| 动态分组 | 成员由规则自动命中，适合沉默召回、高风险观察、开放渠道新增用户等  |

分组管理支持：


| 功能     | 产品规则                           |
| ------ | ------------------------------ |
| 分组统计   | 展示分组总数、静态分组数、动态分组数、启用分组数、今日刷新数 |
| 分组查询   | 按分组名称、分组类型、分组状态、分组来源筛选         |
| 新增分组   | 设置分组名称、类型、说明、成员或规则             |
| 编辑分组   | 修改分组名称、说明、静态成员或动态规则            |
| 启用/停用  | 停用后该分组不再参与运营动作或自动刷新            |
| 批量停用   | 对选中分组批量停用                      |
| 成员详情   | 查看分组内成员、加入时间、加入来源              |
| 静态成员维护 | 支持手动添加、导入、移除成员                 |
| 动态分组刷新 | 按规则重新计算命中成员                    |
| 导出     | 导出当前筛选条件下的分组或成员                |

预设分组：


| 分组名称     | 类型   | 规则或用途                |
| -------- | ---- | -------------------- |
| 重点服务客户   | 静态分组 | 客服或运营手动维护的高价值客户名单    |
| 高风险高活跃用户 | 动态分组 | 风险等级为高，且活跃度为高活跃      |
| 7 日沉默待召回 | 动态分组 | 近 7 日无登录或近 14 日无有效互动 |
| 客服重点回访池  | 静态分组 | 投诉、退款、问题未闭环用户        |

# 六、用户动作

配合用户分群，进行相关一系列与用户的交互效果。

### 6.1 标准流程


| 页面   | 操作           | 结果                |
| ---- | ------------ | ----------------- |
| 用户分群 | 创建或选择一个分群    | 得到目标用户范围          |
| 用户动作 | 新建动作         | 生成一个状态为“未执行”的动作方案 |
| 用户分群 | 点击分群行内“执行动作” | 当前页打开执行动作弹窗       |
| 用户分群 | 选择一个“未执行”动作  | 动作绑定当前分群并执行       |
| 用户动作 | 查看动作列表       | 动作状态进入“执行中”或后续状态  |
| 用户动作 | 查看效果         | 复盘执行人数、排除人数、转化效果  |

### 6.2 页面布局

页面由四部分组成：


| 区域     | 说明                           |
| ------ | ---------------------------- |
| 顶部统计卡片 | 展示动作总数、执行中、待复盘、已完成           |
| 筛选区    | 按动作名称、动作类型、目标分群、动作状态、执行时间筛选  |
| 工具区    | 展示标题“用户动作”和说明，下方左侧展示“新建动作”按钮 |
| 列表区    | 展示动作记录和操作入口                  |

#### 6.2.1 顶部统计卡片


| 卡片   | 统计口径          |
| ---- | ------------- |
| 动作总数 | 当前列表内所有动作数量   |
| 执行中  | 状态为“执行中”的动作数量 |
| 待复盘  | 状态为“待复盘”的动作数量 |
| 已完成  | 状态为“已完成”的动作数量 |

#### 6.2.2 筛选条件


| 筛选项  | 说明                  |
| ---- | ------------------- |
| 动作名称 | 支持按动作名称模糊查询         |
| 动作类型 | 消息触达、权益发放、人工任务、风控处理 |
| 目标分群 | 筛选已绑定过目标分群的动作       |
| 动作状态 | 未执行、执行中、已完成、待复盘、已取消 |

#### 6.2.3 列表字段


| 字段   | 说明                       |
| ---- | ------------------------ |
| 动作名称 | 动作方案名称                   |
| 动作类型 | 消息触达、权益发放、人工任务、风控处理      |
| 目标分群 | 未执行动作显示“-”；已执行动作展示执行目标分群 |
| 执行人数 | 实际或预计执行用户数               |
| 排除人数 | 因账号异常、风险、注销等原因被排除的人数     |
| 状态   | 当前动作状态                   |
| 创建人  | 创建该动作的后台人员               |
| 执行时间 | 未执行动作显示“未执行”；已执行动作展示执行时间 |
| 效果   | 未执行显示“待选择分群执行”；执行后展示复盘指标 |
| 操作   | 查看、编辑、删除、取消              |

#### 6.2.4 动作状态


| 状态  | 含义              | 可执行操作    |
| --- | --------------- | -------- |
| 未执行 | 动作已创建，但尚未绑定分群执行 | 查看、编辑、删除 |
| 执行中 | 动作已绑定分群并开始执行    | 查看、取消    |
| 已完成 | 动作执行完成，效果已统计    | 查看       |
| 待复盘 | 动作已执行完成，但需要运营复盘 | 查看       |
| 已取消 | 动作被人工取消         | 查看       |

#### 6.2.5 新建动作弹窗

新建动作用于创建一个可被分群执行的动作方案。新建动作时不选择目标分群，保存后动作状态为“未执行”。

字段要求：


| 字段   | 是否必填 | 说明                      |
| ---- | ---- | ----------------------- |
| 动作类型 | 是    | 下拉选择消息触达、权益发放、人工任务、风控处理 |
| 创建方式 | 是    | 系统预设模板、自定义动作            |
| 系统预设 | 条件必填 | 选择系统预设模板时必填             |
| 动作名称 | 是    | 默认可由预设模板带出，也可手动修改       |
| 动作内容 | 条件必填 | 自定义动作时必填                |
| 复盘指标 | 否    | 自定义动作建议填写，如回流率、领取率、完成率  |
| 动作备注 | 否    | 说明动作目的、适用场景、注意事项        |

新建动作不应包含：


| 不展示字段 | 原因             |
| ----- | -------------- |
| 目标分群  | 分群应在用户分群页执行时确定 |
| 目标用户  | 目标用户范围来自分群     |
| 排除规则  | 排除规则属于执行动作环节   |
| 动作有效期 | 当前版本暂不要求动作有效期  |

#### 6.2.6 编辑动作

编辑动作用于修改未执行动作。

产品规则：

1. 只有“未执行”动作允许编辑。
    
2. 编辑时回填动作类型、创建方式、系统预设或自定义内容、动作名称、复盘指标和动作备注。
    
3. 已经执行过的动作不允许编辑，避免影响执行记录和复盘数据。
    

#### 6.2.7 删除动作

删除动作用于移除未执行动作。

产品规则：

1. 只有“未执行”动作允许删除。
    
2. 删除前必须二次确认。
    
3. 已执行、执行中、待复盘、已完成动作不可删除。

# 八、活跃度评分

活跃度评分用于衡量用户近期使用深度和关系活跃程度。评分总分 100 分，默认按近 30 日行为计算。分值越高，表示用户越活跃、越稳定。

评分应遵循以下原则：

1. 既关注登录，也关注消息、好友、群聊等有效互动。
    
2. 最近行为权重高于较早行为。
    
3. 只登录但无互动不应被判定为高活跃。
    
4. 存在明显风险行为时，应对活跃评分进行适度扣分，避免高风险用户被误判为优质用户。
    

| 维度   | 分值  | 评分说明                         |
| ---- | --- | ---------------------------- |
| 登录活跃 | 25  | 根据近 30 日登录天数、连续登录天数、最近登录时间评分 |

# 九、风险等级判断

### 9.1 风险评分原则

风险评分用于衡量用户当前存在异常、违规、盗号、骚扰、营销、投诉等风险的可能性。风险评分总分 100 分，分值越高风险越高。

风险等级不仅看分数，也看是否命中强规则。强规则包括黑名单 IP、封禁设备、严重投诉成立、明显违规行为等。

### 9.2 风险评分构成

| 维度   | 分值  | 说明                       |
| ---- | --- | ------------------------ |
| 设备风险 | 25  | 设备复用、频繁换设备、封禁设备再次登录      |
| 登录风险 | 20  | 异地登录、异常 IP、登录失败次数过多、高频登录 |
| 行为风险 | 25  | 高频加好友、高频群发、敏感词命中、异常消息行为  |
| 投诉风险 | 20  | 被举报、被拉黑、投诉成立、客服标记异常      |
| 资料风险 | 10  | 资料缺失、手机号异常、身份信息不一致       |

### 9.3 设备风险判断


| 条件              | 风险分 |
| --------------- | --- |
| 同一设备关联用户数超过平台阈值 | 10  |
| 近 7 日频繁切换设备     | 5   |
| 使用曾被封禁设备登录      | 15  |
| 设备信息缺失或异常       | 5   |

设备风险最高 25 分。命中封禁设备可直接判定为高风险。

### 9.4 登录风险判断


| 条件          | 风险分 |
| ----------- | --- |
| 短时间内跨地区登录   | 8   |
| 登录 IP 命中风险库 | 8   |
| 登录 IP 命中黑名单 | 20  |
| 短时间登录失败次数过多 | 6   |
| 非常用时间段高频登录  | 4   |

登录风险最高 20 分。命中黑名单 IP 可直接判定为高风险。

### 9.5 行为风险判断


| 条件          | 风险分 |
| ----------- | --- |
| 短时间高频加好友    | 8   |
| 短时间发送大量相似消息 | 8   |
| 敏感词命中次数超过阈值 | 8   |
| 群聊刷屏或异常发言   | 6   |
| 被多个用户拉黑     | 5   |

行为风险最高 25 分。

### 9.6 投诉风险判断

| 条件              | 风险分 |
| --------------- | --- |
| 近 30 日被投诉       | 5   |
| 投诉成立            | 10  |
| 近 30 日被举报次数超过阈值 | 8   |
| 客服人工标记异常        | 5   |
| 已有处罚记录          | 8   |

投诉风险最高 20 分。

### 9.7 资料风险判断

| 条件         | 风险分 |
| ---------- | --: |
| 未绑定手机号     |   3 |
| 头像、昵称等资料缺失 |   2 |
| 身份信息不一致    |   5 |
| 注册来源异常     |   3 |


资料风险最高 10 分。

### 9.8 风险等级分层

| 分数或规则                               | 风险等级 | 处理建议                        |
| ----------------------------------- | ---- | --------------------------- |
| 风险评分大于等于 70，或命中封禁设备、黑名单 IP、严重投诉等强规则 | 高风险  | 优先复核，可限制登录、限制加好友、封禁设备或加入黑名单 |
| 风险评分 40-69，存在多项轻中度异常                | 中风险  | 加入观察名单，必要时提醒或限制部分行为         |
| 风险评分小于 40，无明显异常或仅轻微资料缺失             | 低风险  | 正常使用，引导资料补全                 |

# 十一、页面验收标准

### 11.1 用户列表验收

1. 能按关键词、在线状态、用户类型、用户状态、风险等级、标签、分组筛选用户。
    
2. 统计卡片展示用户总量、今日新增、今日登录、当前在线、黑名单用户总数。
    
3. 用户状态、在线状态、登录锁定状态含义清晰，不相互混淆。
    
4. 点击登录日志可查看登录 IP、时间、设备、登录结果。
    
5. 点击用户昵称可进入对应用户详情。
    
6. 批量删除、禁用、拉黑、设备封禁等高影响操作必须有明确确认提示。
    

### 11.2 用户详情验收

1. 能完整展示用户基础资料、账号状态、标签、分组、活跃评分、风险等级。
    
2. 能展示登录设备、最近 IP、好友数、群聊参与、最近行为。
    
3. 能展示该用户的标签摘要、风险摘要和最近事件流。
    
4. 风险等级和活跃等级需与评分规则保持一致。
    
5. 页面能帮助运营、客服、风控判断下一步处理动作。
    

### 11.3 用户画像验收

1. 能按周期、来源、地区筛选画像数据。
    
2. 活跃用户、核心用户、风险用户、资料完整度指标口径清晰。
    
3. 用户分层、生命周期、趋势、关系洞察、风险信号图表均有明确统计含义。
    
4. 风险信号必须展示建议动作，而不只是展示数字。
    
5. 用户明细可追溯到具体用户详情。
    

### 11.4 用户标签验收

1. 标签分类、打标方式、状态、命中用户数展示清晰。
    
2. 人工标签支持手动选择或导入用户。
    
3. 自动标签支持按生命周期、活跃度、风险、账号状态、来源等规则命中。
    
4. 标签停用后不再新增命中，历史记录保留。
    
5. 标签规则必须可解释，命中用户可查看。
    

### 11.5 用户分组验收

1. 静态分组和动态分组定位清晰。
    
2. 静态分组支持人工维护成员。
    
3. 动态分组支持按规则刷新成员。
    
4. 分组详情能展示成员、加入时间、加入来源、规则摘要。
    
5. 分组应能承接运营、客服、风控动作，而不仅是名单展示。
    

### 11.6 风控管理验收

1. 能区分黑名单用户、风险 IP、设备封禁、异常登录、投诉举报等风险对象。
    
2. 风险对象需展示风险来源、风险等级、处理状态和处理建议。
    
3. 高风险对象应优先展示并支持快速处理。
    
4. 解封或解除风险限制时，应记录原因，避免无依据解除。
