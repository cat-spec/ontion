# IM运营管理后台用户风控优化设计方案

## 一、项目概述

当前 `IP黑名单`、`IP风险库`、`设备禁用` 页面以基础列表为主，能完成最小化维护，但缺少风控场景中最关键的“识别 - 处置 - 追溯 - 复盘”闭环。整体目标是把现有页面升级为统一的用户风控管理模块，提升管理员的筛查效率、处置效率和审计能力。

优化目标：å
- 统一三类风险对象的管理入口和视觉规范。
- 补齐风险等级、命中次数、有效期、来源、影响范围等核心信息。
- 支持批量维护、导入导出、详情追溯和操作审计。
- 降低误封风险，建立白名单、到期自动解封和二次确认机制。

## 二、功能范围

- 统一风控中心导航与页签结构。
- 顶部概览卡片：总量、今日新增、待处理、命中次数。
- 高级筛选：关键词、风险等级、状态、来源、创建人、时间范围。
- 列表优化：固定表头、固定操作列、状态标签、批量操作。
- 详情抽屉：基础信息、风险记录、关联用户/设备、操作日志。
- 新增/编辑/导入：单条新增、批量导入、模板下载、校验提示。
- 自动化策略：到期自动解封、规则升级、白名单豁免、风险联动。
- 审计能力：操作原因、操作人、操作时间、变更记录可追踪。

## 三、核心模块设计

### 3.1 页面整体布局

- 左侧保持现有后台导航，用户管理下新增“风控中心”总入口，三页统一为 `IP黑名单 / IP风险库 / 设备禁用` 页签。
- 顶部增加概览区，展示总量、今日新增、待处理、命中次数，替代当前纯列表式开局。
- 筛选区固定在页面顶部，支持展开高级筛选，避免条件过多时挤压表格。
- 列表区保留主表，但操作列固定在右侧，批量操作放到表格上方。
- 详情采用右侧抽屉，不跳页查看，减少上下文丢失。

### 3.2 IP黑名单

- 支持单 IP、IP 段、CIDR、IPv6。
- 表字段建议扩展为：IP、封禁范围、风险等级、有效期、命中次数、最近命中时间、创建人、创建时间、状态、备注。
- 行内操作建议为：编辑、停用、延长、删除；高危动作需二次确认并填写原因。
- 新增时校验与风险库、白名单、已有黑名单的冲突关系。

### 3.3 IP风险库

- 明确其定位为“风险情报库”，不是直接封禁库。
- 表字段建议扩展为：IP、风险等级、来源、可信度、处置建议、最近命中时间、关联账号数、创建人、创建时间、状态。
- 支持“加入黑名单”“加入白名单”“标记已核实”“批量导入”。
- 对高风险 IP 提供自动升级规则，例如命中阈值达到后自动进入黑名单。

### 3.4 设备禁用

- 先明确设备标识类型：deviceId、IMEI、OAID、IDFA、Android ID 或自研指纹。
- 表字段建议扩展为：设备标识、设备名称、品牌、型号、禁用原因、有效期、命中次数、关联用户数、封禁时间、状态。
- 空态改为操作引导，提供“新增禁用设备”“批量导入”“从用户详情加入禁用”三个入口。
- 支持从用户详情页反向定位设备，方便客服快速处置。

### 3.5 统一样式与交互

- 风险类标签统一使用色阶：低风险灰/蓝，中风险橙，高风险红，已处理绿。
- 主按钮保持“新增/导入/批量处置”三类，危险按钮统一红色并下沉到二级菜单。
- 表格建议支持密度切换、固定首列/尾列、空状态插图、分页跳转和横向滚动提示。
- 所有新增和删除类操作都需要操作原因、操作者、时间写入审计日志。

## 四、数据结构与流向

- `risk_ip_blacklist`：`id`、`ip_value`、`ip_type`、`ban_scope`、`risk_level`、`expire_at`、`hit_count`、`last_hit_at`、`status`、`remark`、`created_by`、`created_at`、`updated_by`、`updated_at`
- `risk_ip_pool`：`id`、`ip_value`、`risk_level`、`source_type`、`credibility`、`suggested_action`、`related_account_count`、`last_hit_at`、`status`、`remark`
- `device_blacklist`：`id`、`device_key`、`device_type`、`device_name`、`brand`、`model`、`reason`、`expire_at`、`hit_count`、`related_user_count`、`status`
- `operation_log`：`id`、`biz_type`、`biz_id`、`action_type`、`reason`、`operator`、`operate_time`、`before_data`、`after_data`

数据流建议：
- 列表页查询 -> 返回分页数据 + 汇总指标。
- 新增/编辑 -> 表单校验 -> 落库 -> 刷新列表 -> 记录审计日志。
- 命中事件 -> 写入风险库命中记录 -> 触发规则判断 -> 可能升级为黑名单或设备禁用。
- 批量导入 -> 文件解析 -> 逐条校验 -> 输出成功/失败明细 -> 支持失败重试。

## 五、边界情况处理

- IP 重复、网段重叠、IPv4/IPv6 混填时必须提示冲突。
- 黑名单与白名单冲突时，优先级需明确，建议白名单仅对特定动作豁免并保留审计。
- 设备标识不完整时，不允许直接禁用，应提示补充来源或使用指纹补全。
- 批量导入失败时，需返回失败原因和行号，避免整批回滚导致不可用。
- 删除建议改为停用/归档，防止历史证据丢失。
- 权限控制需区分查看、编辑、导入、导出、删除、审批五类权限。
