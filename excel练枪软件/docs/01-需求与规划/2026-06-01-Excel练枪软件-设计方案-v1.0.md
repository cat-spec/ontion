# Excel练枪软件设计方案

## 一、项目概述

本项目目标是基于 Excel 和 VBA 宏实现一个可直接交互的练枪小游戏。游戏以单元格文字渲染人物目标，用户通过点击单元格来完成定位和射击，系统在限定时间内持续刷新随机人形和指定命中部位，并实时完成判定、计分与结算。

核心价值：
- 用 Excel 即可运行，不依赖额外客户端。
- 通过文字人形实现轻量化“目标识别 + 定位点击”训练。
- 利用随机模板与指定部位规则，提高反应和精准点击能力。

## 二、功能范围

- 开始游戏：点击按钮后开始一局限时训练。
- 限时连打：默认单局 `30 秒`，在时限内连续刷新目标。
- 随机人形：从预置的多套文字人形模板中随机抽取。
- 指定部位：每回合随机要求命中 `头 / 胸 / 腹 / 腿`。
- 点击判定：用户通过点击单元格完成击发，系统判断是否击中目标部位。
- 即时反馈：显示命中、误击、脱靶、连击等反馈信息。
- 成绩统计：记录总分、总点击数、有效命中数、错误点击数、命中率和最高连击。
- 结束结算：时间结束后自动停止并展示本局成绩。

## 三、核心模块设计

### 3.1 工作簿结构

- `开始页`：标题、玩法说明、开始按钮。
- `游戏页`：主靶区、状态栏、即时反馈区。
- `结算页`：本局结果统计与重开按钮。
- `配置页`：模板定义、分值规则、基础配置参数。

### 3.2 游戏交互流程

1. 用户在 `开始页` 点击开始按钮。
2. 系统初始化分数、时间、连击、点击次数等状态。
3. 随机抽取一套文字人形模板。
4. 随机抽取本回合要求命中的部位。
5. 将人形渲染到 `游戏页` 靶区，并显示目标部位。
6. 用户点击单元格。
7. 系统判断点击位置：
   - 是否属于当前人形；
   - 是否属于当前要求命中的指定部位。
8. 依据结果更新分数、命中统计和反馈文案。
9. 立即刷新下一回合目标。
10. 倒计时归零后结束游戏并跳转到 `结算页`。

### 3.3 文字人形模板设计

第一版采用多套预置模板而不是动态拼接，既保证“随机人形”效果，也便于稳定判定。每个模板包含两类信息：

- 字符布局：哪些单元格显示头、躯干、四肢相关字符。
- 部位映射：哪些单元格属于 `头 / 胸 / 腹 / 腿`。

模板建议首版提供 `6` 套：
- 标准站姿
- 微侧身
- 手臂上抬
- 半蹲姿态
- 宽站姿
- 紧凑站姿

### 3.4 判定与计分规则

- 命中指定部位：`+10`
- 命中人物但非指定部位：`-2`
- 点击空白区域：`-1`
- 连续正确命中达到每 `5` 次：额外 `+5`

判定结果需同步更新：
- 当前分数
- 总点击数
- 有效命中数
- 错误点击数
- 当前连击
- 最高连击
- 最近一次反馈文案

### 3.5 模块划分

- `GameState`：管理当前游戏状态、时间、分数、统计数据。
- `TemplateLibrary`：管理所有人物模板和部位坐标映射。
- `Renderer`：负责清空靶区、绘制当前模板、更新界面状态栏。
- `Judge`：根据点击单元格判定命中情况并返回结果。
- `Bootstrap`：负责开始游戏、结束游戏、切页、按钮绑定。

## 四、数据结构与流向

### 4.1 核心状态

- `IsGameRunning`：是否处于游戏中
- `GameDurationSeconds`：单局时长
- `GameStartTime`：开始时间
- `Score`：当前分数
- `Shots`：总点击数
- `Hits`：有效命中数
- `Misses`：错误点击数
- `Combo`：当前连击
- `BestCombo`：最高连击
- `CurrentTemplateId`：当前模板编号
- `CurrentTargetPart`：当前要求命中的部位

### 4.2 模板数据

每套模板包含：
- `TemplateId`
- `DisplayCells`
- `HeadCells`
- `ChestCells`
- `BellyCells`
- `LegCells`

数据流如下：

1. 开始按钮触发初始化。
2. 系统根据模板库生成当前回合目标。
3. 渲染模块写入表格。
4. 用户点击靶区后，判定模块读取坐标并返回结果。
5. 状态模块更新统计数据。
6. 渲染模块刷新分数栏和提示文本。
7. 倒计时结束后输出到 `结算页`。

## 五、边界情况处理

- 非游戏状态点击靶区：忽略，不计分。
- 点击按钮区、信息栏或非靶区：忽略，不计失误。
- 倒计时结束与点击同时发生：优先按时间结束处理，避免多计一次。
- 同一回合多次点击：第一次点击完成判定后立即进入下一回合。
- 模板刷新前后误点：通过回合锁控制，只允许当前回合的第一次点击生效。
- Excel 缩放导致变形：固定列宽与行高，保证文字人形布局稳定。
- 宏未启用：在开始页提示启用宏，否则无法开始游戏。

## 六、第一版实现边界

第一版只实现以下内容：
- `30 秒` 限时连打
- `6` 套随机文字人形
- `4` 个部位判定：头、胸、腹、腿
- 开始、结算、重开
- 即时反馈与基础计分

暂不实现以下内容：
- 排行榜
- 多难度模式
- 闯关模式
- 音效
- 动态移动目标
- 历史记录持久化
