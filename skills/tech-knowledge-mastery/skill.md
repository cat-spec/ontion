# 技术知识精要生成器

## 技能概述

当用户询问某项技术时，自动生成该技术的全面知识点文档。不同于简单的文档摘要，本技能专注于：

1. **知识连贯性**：按学习路径组织，确保知识不跳跃
2. **高度概括**：用自己的话解释，不照搬官网
3. **多来源题目**：面试/论坛/期末/官网/实战，每章≥10道
4. **实战导向**：项目常用场景 + 陷阱提示 + 实践信号
5. **深度挑战**：费曼总结 + 3个综合实践 + 5个最难问题

---

## Agent 协作架构

```
tech-knowledge-mastery skill（主流程：调度 + 文件写入）
  ├── doc-collector agent（文档收集者）
  │     职责：定位并收集官方文档、API参考、核心社区资源
  │     输出：文档链接列表、章节结构概览
  │
  ├── knowledge-organizer agent（知识组织者）
  │     职责：分析文档结构，按学习路径重组章节地图
  │     约束：确保知识连贯不跳跃
  │
  ├── content-generator agent（内容生成者）
  │     职责：逐章生成知识点、题目、场景、陷阱、实践信号
  │     约束：每章≥10道题目，来源多样化
  │
  └── document-writer（主流程）
        职责：格式化输出 MD 文档
```

## Step A: 生成模式

### A1: 技术识别与信息收集

首先识别用户询问的技术，并收集必要的上下文信息：

**必须收集**：
- 技术名称（如：React、Vue、TypeScript）
- 目标版本（如：React 18，如用户未指定则使用当前稳定版）
- 输出路径（可选，未指定则使用默认路径）
- 学习者水平（可选：初学者/中级/高级）

**信息来源优先级**：
1. 官方文档（官方文档站、官方 API 参考、官方指南）
2. 官方 GitHub 仓库（README、Issues、Discussions）
3. 核心社区资源（Stack Overflow 高赞问题、技术博客）

### A2: 文档收集与章节地图构建

启动 `doc-collector` 和 `knowledge-organizer` agent（可并行）：

**doc-collector 任务**：
```yaml
输入: 技术名称、目标版本
输出:
  - 官方文档主页 URL
  - API 文档 URL
  - 官方教程/Guide URL
  - 官方 GitHub 仓库 URL
  - 3-5 个核心社区资源链接
```

**knowledge-organizer 任务**：
```yaml
输入: 文档结构分析结果
输出:
  - 章节列表（按学习路径重组）
  - 每章的核心知识点概述
  - 章节间的依赖关系
```

**知识地图示例**：
```
1. 技术定位与核心模型
2. 环境与快速开始
3. 基础语法/核心概念
4. 组件/模块系统
5. 状态管理/数据处理
6. 路由/导航
7. 高级特性与性能优化
8. 生态系统与工具链
9. 最佳实践与常见模式
10. 测试与调试
```

### A3: 生成任务清单

**必须先展示任务清单，再逐个生成。**

任务拆分原则：**每个主要章节是一个独立任务。**

任务清单示例：
```
| 序号 | 任务名称 | 类型 | 状态 |
|------|---------|------|------|
| 1  | 文档头部信息 | [文档] | [ ] 等待中 |
| 2  | 知识体系总览 | [文档] | [ ] 等待中 |
| 3  | 第1章：技术定位与核心模型 | [Agent] | [ ] 等待中 |
| 4  | 第2章：环境与快速开始 | [Agent] | [ ] 等待中 |
| 5  | 第3章：基础语法/核心概念 | [Agent] | [ ] 等待中 |
| ... | ... | ... | ... |
| N  | 费曼总结 | [文档] | [ ] 等待中 |
| N+1| 综合实践问题（3题） | [文档] | [ ] 等待中 |
| N+2| 该领域最难挑战（5题） | [文档] | [ ] 等待中 |
```

### A4: 逐章内容生成

对每个 [Agent] 任务，启动 `content-generator` agent：

**content-generator 输入**：
```yaml
输入:
  - 技术名称和版本
  - 当前章节名称和范围
  - 官方文档链接
  - 章节核心知识点概述

输出:
  - 核心知识点（高度概括，非照搬）
  - 章节题目（≥10道，多来源）
  - 项目常用场景
  - 易混淆知识点
  - 常见陷阱与坑点
  - 实践信号（官方进阶链接 + 社区热议 + 动手验证）
```

### A5: 章节题目生成规范

每章必须生成≥10道题目，来源必须多样化：

**题目来源要求**：
- 至少 2 道面试题（公司面试、技术面试真题）
- 至少 2 道论坛题（Stack Overflow 高赞、知乎热帖、技术社区）
- 至少 2 道期末题/认证题（大学课程、认证考试）
- 至少 2 道官网题（官方 FAQ、官方练习）
- 至少 2 道实战题（项目开发场景）

**题目难度分层**：
- 基础题（3-5道）：考察基本概念和语法
- 进阶题（3-5道）：考察深入理解和应用
- 深度题（2-5道）：考察原理和底层机制
- 实战题（2-3道）：考察项目场景应用

### A6: 实践信号要求

每个章节必须包含实践信号部分：

**实践信号结构**：
```markdown
### 实践信号

#### 官方进阶文档
- **[主题名称]**：[链接] - 学习重点：[具体说明]

#### 社区热议话题
- **[话题名称]**：来源于 [来源] - 讨论要点：[核心观点]

#### 动手验证
请完成以下实践任务：
1. [具体的、可执行的实践任务1]
2. [具体的、可执行的实践任务2]
```

### A7: 文档输出

所有章节生成完成后，调用 `document-writer`（主流程）输出完整 MD 文档。

**输出路径规则**：
- 用户指定路径：使用用户指定路径
- 未指定路径：`{技术名称}-知识精要-{YYYYMMDD}.md`

## 关键约束

1. **知识连贯性**：按学习路径组织章节，确保前后连贯，不跳跃
2. **概括性**：每个知识点用自己的话概括，不照搬官网原文
3. **题量保证**：每章≥10道题目，来源必须多样化
4. **实践导向**：每个实践信号必须包含具体可执行的动手任务
5. **深度挑战**：最后的5个最难问题必须是该领域真正深度的挑战，不是常见题

---

## 质量检查清单

输出文档必须包含以下元素：

- [ ] 文档头部：技术名称、版本、来源、生成时间
- [ ] 知识体系总览：章节导航
- [ ] 每章包含：核心知识点、≥10道题目、项目场景、易混点、陷阱、实践信号
- [ ] 费曼总结：用简单语言重述核心概念
- [ ] 3个综合实践问题：跨章节综合应用
- [ ] 5个最难挑战：该领域深度问题
