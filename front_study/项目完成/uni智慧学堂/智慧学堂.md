## 一、项目定位

- **应用名称**：（待定，如“智慧课堂学生端”）
    
- **目标用户**：在校学生（K12、职校或高校）
    
- **核心价值**：查看课程资源、完成学习任务、接收通知、提交作业/测验、查看学习进度。
    
- **技术约束**：UniApp 打包为 Android APK，前台展示（不涉及教师端复杂管理），需适配手机/平板。
    

---

## 二、用户角色与用例

|角色|核心用例|
|---|---|
|学生|登录/注册、浏览课程、观看视频/图文、做测验、提交作业、查看成绩与反馈、接收公告、修改个人信息|

> 注：教师端后台数据由另一个系统（或管理后台）提供，本 APK 仅消费 API。

## 三、功能需求（详细）

### 1. 用户认证模块

|功能点|说明|
|---|---|
|账号密码登录|支持手机号/学号 + 密码|
|短信验证码登录|可选，需集成短信服务|
|自动登录|Token 存储，7 天有效|
|忘记密码|通过手机号重置|
|退出登录|清除本地缓存，返回登录页|

### 2. 首页（学习中心）

|功能点|说明|
|---|---|
|轮播图|展示课程活动、重要通知|
|学习进度概览|显示“进行中课程数/总课程数”、“未读通知数”|
|常用入口|我的课程、最新通知、待办作业、测验入口|
|推荐课程|根据学生专业/年级推荐 3~5 门课|

### 3. 我的课程

|功能点|说明|
|---|---|
|课程列表|展示所有已选课程（封面、名称、进度条）|
|课程分类|按学期、状态（进行中/已结束）筛选|
|课程详情|包含课程简介、教师、教学大纲|
|课程章节|树形结构：章 → 节 → 学习单元（视频/文档/测验）|

### 4. 学习单元（内容消费）

|类型|功能|
|---|---|
|视频|支持播放/暂停/进度条/倍速，记录上次播放位置，观看时长计入完成度|
|图文|富文本渲染（图片、表格、代码块），可缩放|
|PDF/课件|在线预览（使用 uni.downloadFile + 系统预览或嵌入 PDF 插件）|
|外部链接|WebView 内打开，需返回按钮|

### 5. 作业与测验

|功能点|说明|
|---|---|
|作业列表|待提交 / 已提交 / 已批阅三个标签|
|作业详情|题目描述、附件、截止时间、提交状态|
|作业提交|文本输入 + 上传图片/文件（调用相机/相册）|
|测验|单选题、多选题、判断题（支持自动评分）|
|成绩查看|作业得分、测验分数、教师评语|

### 6. 通知公告

|功能点|说明|
|---|---|
|消息中心|按时间倒序，区分“系统通知”与“课程通知”|
|已读未读|未读红点标记|
|详情页|富文本展示内容，可包含附件链接|

### 7. 个人中心

|功能点|说明|
|---|---|
|个人信息|头像（上传/拍照）、昵称、学号、班级|
|学习统计|总学习时长、完成作业数、平均测验正确率|
|修改密码|旧密码 + 新密码|
|关于我们|版本号、用户协议、隐私政策|
|意见反馈|文本 + 图片，提交到后台|

## 四、非功能需求

|类别|要求|
|---|---|
|性能|首页加载 ≤ 1.5 秒（4G 环境），视频起播 ≤ 2 秒|
|兼容性|Android 8.0 ~ 14，主流分辨率（720p 以上）|
|离线缓存|课程图文、已下载的视频可离线观看（可选）|
|安全性|密码 RSA 加密传输；Token 存至 `uni.setStorageSync`；敏感 API 加时间戳签名|
|易用性|按钮点击区域 ≥ 44px，视频支持全屏手势，字号可跟随系统|
|网络|无网络时友好提示，自动重试；支持 HTTPS|

## 五、数据结构简表（关键字段）

**课程 Course**

- id, name, cover, teacher, term, status(0-未开始,1-进行中,2-已结束), progress
    

**章节 Chapter**

- id, course_id, title, sort
    

**学习单元 Lesson**

- id, chapter_id, type(video/text/pdf/quiz), title, content_url, duration, is_free
    

**作业 Homework**

- id, course_id, title, description, deadline, attachment_url, submit_type(text/file)
    

**提交记录 Submission**

- id, homework_id, student_id, content, files, score, feedback, submit_time
    

**测验 Quiz**

- id, lesson_id, questions (JSON: 题干,选项,答案,分值)

## 七、技术实现建议（UniApp 侧重点）

1. **状态管理**：使用 `pinia`  存储用户信息、Token、课程进度。
    
2. **网络请求**：封装 `uni.request`，统一处理 loading、错误码（401 跳转登录）。
    
3. **视频组件**：推荐使用 `video` 原生组件，记录播放进度需监听 `@timeupdate` 并定时上报。
    
4. **富文本**：使用 `rich-text` 或 `uView` 的 `u-parse` 组件。
    
5. **文件上传**：调用 `uni.chooseImage` + `uni.uploadFile`。
    
6. **本地存储**：课程列表缓存 5 分钟，减少接口请求。
    
7. **打包 APK**：使用 `HBuilderX` → 发行 → 原生App-云打包，配置应用图标、权限（摄像头、存储、网络）。
