# Is It? 微信小程序需求分析

基于是非问答的 AI 猜谜游戏。用户无需注册，即开即玩，通过连续提出“是/不是”问题推理秘密事物并完成猜测。

MVP 4 周

首版交付周期

3 步

核心体验路径

≤ 3 秒

目标单次回答耗时

≥ 30%

局内完成猜中率目标

---

## 1. 产品目标与范围

喜欢逻辑推理、知识挑战、轻量休闲互动的普通用户。

使用微信生态，偏碎片化时间游玩，期望快速进入和即时反馈。

验证“AI 是非猜谜”玩法是否具备持续吸引力与分享传播潜力。

在首版仅聚焦单人闯关体验，不引入社交对战和复杂成长系统。

---

## 2. 核心玩法与业务规则

| 模块    | 规则定义                   | MVP 约束              |
| ----- | ---------------------- | ------------------- |
| 题目生成  | AI 预设一个秘密事物（如动物、物品、地点） | 优先常见实体；过滤歧义和冷门词     |
| 提问机制  | 用户每次输入一个可回答“是/不是”的问题   | 非是非问句触发引导改写         |
| AI 回答 | 仅返回“是”或“不是”及简短提示       | 禁止泄露答案本体；默认 1-2 句说明 |
| 猜测提交  | 用户可随时提交“我猜是 X”         | 支持模糊同义判断；失败继续问答     |
| 胜负判定  | 猜中即通关；超上限未猜中判定失败       | 默认 20 问上限，可在配置调整    |
| 局后反馈  | 展示答案、关键推理路径、鼓励再来一局     | 保存最近战绩，支持一键再开       |

---

## 3. 用户流程（主链路）

| 阶段   | 用户行为        | 系统行为          | 关键体验点    |
| ---- | ----------- | ------------- | -------- |
| 进入首页 | 点击“开始游戏”    | 创建新对局并生成秘密事物  | 无登录、秒开局  |
| 问答推进 | 输入是非问题并查看回答 | 校验问题类型并返回是/不是 | 低时延、反馈稳定 |
| 发起猜测 | 输入猜测词条      | 执行同义词与近义匹配判定  | 判定透明、可解释 |
| 结束复盘 | 查看结果并选择再来一局 | 展示答案与回合数据     | 失败也有收获感  |
| 分享传播 | 分享小程序给好友    | 携带活动或战绩参数     | 形成自然拉新闭环 |
|      |             |               |          |

---

## 4. 功能需求拆解（MVP）

1) 首页：玩法说明、设置两个难点，猜人，和猜物。

2) 对局页：问题输入、消息流、剩余提问次数、猜测入口。

3) 结果页：是否猜中（猜中弹出奖杯动画）、标准答案、回合摘要、再来一局。

4) 反馈入口：问题举报与建议提交。

5) 题库与动态生成：按分类控制难度和词汇安全。

6) 问句判定：识别非是非问句并提供改写建议。

7) 回答策略：严格“是/不是”主回答，附最小提示。

8) 猜测判定：同义词词典 + 语义相似度阈值。

9) 风控审计：敏感词过滤、异常请求限流。
